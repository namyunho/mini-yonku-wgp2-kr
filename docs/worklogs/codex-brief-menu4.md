# Codex 배치 브리핑 — System ④ 메뉴 소형폰트 라벨 한글화 (월드맵 X메뉴·조작방법)

> 작성: Claude, 2026-07-21. 규명·설계·정본 = **docs/18**(먼저 읽을 것) + docs/11(폰트/렌더러)·docs/12(SJIS 한글 인프라)·`assets/translation_guide/glossary.md`(용어). **정본 코드/커밋은 Claude가 소유**(handoff 규약). Codex는 아래 스펙+게이트 범위만 수행하고 산출을 Claude에 반환한다.

## 배경 (규명 완료 사실)

- 대상 = **System ④**: 직접 타일 렌더러 **`$C0:1B4B`** + **$D9 8×8 2bpp 폰트**(VRAM 타일256, docs/11). SJIS도 $CA도 아님.
- 렌더러 `$C0:1B4B`(디스어셈블 완료): 문자열 1바이트씩 → `tile = byte + base`(base=DP `$00` 상위, 예 0x2100=VRAM256|pal|prio) → 타일맵버퍼 `$7E:0000,X`. `0x00`=메인행 종료→오버레이행(`$7D:FFC0,X`, 탁점; 제어 jmp표 `$C0:1BB0`의 idx 0/1/2). `0xFF`=빈타일(→0).
- **호출부 10곳**(JSL `$C0:1B4B` = `22 4B 1B C0`): `$C0:719A · $C0:778C · $C0:79F9 · $C0:7A0D · $C0:7A31 · $C0:7C00 · $C0:8D6F · $C3:8B81 · $C3:95A4 · $C3:99B8`. 호출자가 문자열 24bit 포인터를 `$7E:000E`에, dest X(타일맵 오프셋)를 설정.
- **라벨 문자열표**(직접 타일인덱스, 메뉴 스크립트 = 오프코드+인라인 문자열 혼합):
  - **$C7:B180~B460** = 조작방법 튜토리얼 8페이지(목차·버튼캡션·파라미터·모드타이틀). ⚠️ 스크립트 오프코드 섞임(0x05·0x94·0x97·페이지참조 등) — 예: もくじ=$C7:B188 바이트 `23 08 0c`가 오프코드 사이 인라인.
  - **$C1:C6D0~C7A0** = 세팅 창메뉴(박스문자 $E0~$E7 포함).
  - **$CE:46xx~** = 개러지 메뉴(そうさ 등).

### 타일인덱스 ↔ 문자 맵 (변환표 $C1:D1C3 역산)

```python
rom = open("roms/Mini Yonku Let's & Go!! - Power WGP 2 (J) (NP).smc","rb").read()
TBL=0x01D1C3
def r16(o): return rom[o]|(rom[o+1]<<8)
def tile_of(hi,lo):            # SJIS(hi,lo) → (dakuten<<8)|body_tile
    return r16(TBL + r16(TBL+(hi&0x7F)*2) + (lo-0x40)*2)
# 카나/라틴/숫자 문자를 cp932로 인코딩→tile_of→ body=val&0xFF 로 char↔tile 맵 구성.
# body만 메인행 타일. 탁점(dakuten=val>>8)은 오버레이행($7D:FFC0)에 별도.
```

## 한글 렌더 방식 (사용자 확정)

docs/12가 **8pt 한글 200음절을 VRAM 800~999**에 올리고 SJIS 렌더러 $C1:965E에 **0xFE 마커 훅**($C1:9843): 표값 상위==0xFE면 tile=low+0x220+base=VRAM800+low. ④에도 **동형 마커 훅을 $C0:1B4B에 이식**(Claude 담당):
- 메인행 루프에서 `byte==0xFE`면 다음바이트=한글 음절 idx → `tile = idx + 0x220 + base`(VRAM800+idx), 오버레이 없음.
- 음절 풀 = build_sjis.py의 `syls`(SJIS UI 200) **뒤에 라벨 순증 음절 append**. **VRAM 상한 224**(현 200 + 순증 ≤24). 초과 금지.

### ⚠️ 핵심 과제 (Codex가 풀 것): 바이트길이 문제

직접 렌더러는 **1바이트=1타일**이라 마커 한글은 **2바이트/음절**로 원본보다 길어짐(예 "스피드성능" 5음절=10B > "スピードせいのう" 8타일=8B). 각 라벨 슬롯(스크립트 내 span)에 **in-place로 안 들어갈 수 있음**. 표별로 전략 결정:
1. **in-place 우선**: 한글이 슬롯 byte 이하면 마커 인코딩 그대로 채우고 남으면 빈타일(0xFF) 패딩.
2. **초과 시**: (a) 의미보존 축약으로 byte 맞추기, 또는 (b) **base-스왑 1바이트 한글**(순수 한글 라벨 한정 — 그 라벨 draw의 base를 VRAM800 영역으로 바꾸면 1바이트 idx로 한글 도달, 마커 불필요 → byte 절반). base는 호출자가 전달하므로 (b)는 호출부/스크립트 레코드 수정 필요 → **호출부 RE 결과에 따라 Claude와 협의**.
3. 스크립트 오프코드·좌표·오버레이·종료바이트는 **절대 보존**(문자열 타일만 교체).

## Codex 작업 (순서)

1. **호출부 RE**: JSL `$C0:1B4B` 10곳 disasm → 각 호출자가 (a) 문자열 포인터를 어떻게 세팅하는지($7E:000E), (b) base(pal/prio/tilebase)를 무엇으로 넘기는지, (c) 스크립트 표($C7:B180 등)를 어떻게 순회하는지 규명. idalib-mcp 사용(docs/16 §6, .i64 = roms/…smc.i64).
2. **스크립트 포맷 파서**: $C7:B180~B460·$C1:C6xx·$CE:46xx의 오프코드/좌표/문자열 span을 분해하는 파서 작성. 각 라벨의 **정확한 byte span**(시작·끝, 오버레이 포함)을 추출.
3. **재인코딩**: 위 번역표대로 각 라벨의 메인행 문자열 타일을 (0xFE, kor_idx) 마커열로 교체. 오프코드·좌표·오버레이·종료 보존. byte-fit 검사 → in-place 가능/축약/ base-스왑 후보를 표별로 판정·플래그.
4. **산출**: JSON `assets/translations/menu4_labels.json` = 배열 of `{table, addr, jp, kr, orig_span_hex, new_bytes_hex, mode: inplace|abbrev|baseswap|overflow, note}` + 순증 음절 리스트. + 호출부 RE 요약 md.

## 게이트 (gate↔brief 락스텝 — 아래 전부 Codex 자체검사 + Claude 재검)

1. **디코드 일치**: new_bytes의 메인행 마커열을 kor_idx→음절로 디코드 → kr와 **정확히 일치**.
2. **오프코드/구조 보존**: 문자열 타일 외 모든 바이트(0x05/0x94/0x97/좌표/오버레이/0x00 종료·박스 $E0~$E7)가 원본과 **완전 동일** 위치.
3. **byte-fit**: new span byte ≤ orig span byte(in-place). 초과분은 mode=abbrev/baseswap/overflow로 **명시 플래그**(임의 절단 금지).
4. **음절 상한**: 순증 음절 + SJIS UI 200 ≤ **224**. 초과 시 중단·보고.
5. **폰트 커버리지**: 모든 한글 음절이 `assets/fonts/small/*.bin`(glyph_map)에 존재(없으면 거부).
6. **누락 0**: 위 번역표 전 항목 산출.
