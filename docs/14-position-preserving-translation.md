# 14 · 위치보존 번역 — 어드벤처 VM 크래시 원천 차단 계획 (SSOT)

## 문제 정의 (확정)

어드벤처 씬 번역이 **런의 바이트 길이를 바꿔(위치 밀림)**, 위치에 의존하는 VM 로직
(cmd 0x20 컨테이너·cmd 0x54 조건분기·루프)이 옛 위치를 가리켜 **리셋/행/프리즈**를 일으킨다.
- `round-trip`은 압축 무손실만 증명하지 위치 정합은 보장 못 한다.
- **정적 탐지 실패**: 조건분기(cmd 0x53/0x54) 씬이 86개인데 그중 어느 게 깨질지 구별 신호가 없다
  (offset 명령 hit비율 전부 낮음, cond 워드도 위치 불일치). 개별 반응적 대응은 시간 낭비.

## 감사 결과 (2026-07-19, `roms` 원본 vs `adventure_kr.json`)

| 지표 | 값 |
|------|-----|
| 번역 cmd 0x21 런 | 1686 |
| **길이 불일치(밀림) 런** | **1490 (88%)** ← 버그 원천 |
| 한글이 원본보다 **긴** 런(패딩 불가) | **447 (27%)** ← 축약 재번역 대상 |
| 한글이 짧거나 같은 런(패딩 가능) | ~1239 |

delta 분포: 대부분 −1~−4(한글 짧음)이나 +1~+5(한글 김)도 상당수.

## 근본 예방 = **위치보존 번역** (사용자 확정 방침)

1. **한글이 짧은 런** → **원본 바이트 길이로 자동 패딩**. 공백 글리프(전각공백 idx=1→`0x11`,
   반각공백 idx=0→`0x10`, **둘 다 1바이트**라 어떤 delta든 정확히 채움)를 **런 말미**(마지막 제어코드
   뒤, `0x00` 앞)에 삽입 → 위치 100% 보존 → VM 로직 안 깨짐.
   - ⚠️ **표시 안전성 미검증**: 말미 공백이 대화상자에서 안 보이는지 **0xB0으로 프로토타입 실기 확인 필수**.
     (말미 blank는 좌측정렬 대화상자에서 대개 안 보이나, {clear} 뒤 처리 등 실측 필요.)
2. **한글이 긴 런** → **의미 보존하며 원본 바이트 길이 이하로 축약 재번역**. 축약 전까지 그 씬은
   원본유지(안전). → 이 447런이 "재번역 목록"(추려서 처리).
3. **씬 단위 규칙**: 씬의 모든 번역런이 원본 이하(패딩 가능)면 → 번역+패딩(안전). 하나라도 긴 런이면
   → 원본유지 + 축약목록에 등재. 축약 후 재빌드하면 그 씬도 안전 번역됨.
4. **cmd 0x20 컨테이너 씬(16개)**: 별도(컨테이너 포맷 RE 후 내부 0x21런만 치환). 현재 원본유지.

→ 결과: **밀린 위치가 ROM에 절대 안 실림 → 프리즈/리셋 원천 차단 → 안정 ROM으로 QA/디버그 가능.**

## 구현 계획 (build_adv.py)

- `encode_text(kr)` 결과가 원본 텍스트 길이보다 **짧으면** 말미에 공백글리프로 패딩해 정확히 일치시킴.
  cmd 0x21 런: `out += buf[prev:at+1]` + `encode_text(kr)` + **패딩** + `b"\x00"`, 총 길이 = 원본 런 길이.
- **긴 런 탐지**: `len(encode_text(kr)) > 원본텍스트길이`면 그 씬을 `LONGER_REVERT`(원본유지) + 리포트.
- 빌드 산출: **축약 필요 런 목록**(scene, at, 원본len, 한글len, 초과바이트, text_kr) → `out/retranslate_longer.json`.
- 검증: 각 씬 build 후 walk 구조 동일 확인(현행) + **Mesen 실기**(round-trip≠VM유효).

### 0xB0 프로토타입 (검증 먼저)
0xB0은 23런 중 **긴 런 3개**(@0x020A +4, @0x0369 +3, @0x0443 +5). 이 3런만 축약하면 나머지는 패딩 가능.
→ 3런 축약 후 전체 패딩 → 실기로 **프리즈 소멸 + 번역 유지** 확인 → 방식 검증 완료 → 전 씬 적용.
