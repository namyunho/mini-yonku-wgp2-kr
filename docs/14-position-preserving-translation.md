# 14 · 위치보존 번역 — 어드벤처 VM 크래시 원천 차단 계획 (SSOT)

## 문제 정의 (확정)

어드벤처 씬 번역이 **런의 바이트 길이를 바꿔(위치 밀림)**, 위치에 의존하는 VM 로직
(cmd 0x20 컨테이너·cmd 0x54 조건분기·루프)이 옛 위치를 가리켜 **리셋/행/프리즈**를 일으킨다.
- `round-trip`은 압축 무손실만 증명하지 위치 정합은 보장 못 한다.
- **정적 탐지 실패**: 조건분기(cmd 0x53/0x54) 씬이 86개인데 그중 어느 게 깨질지 구별 신호가 없다
  (offset 명령 hit비율 전부 낮음, cond 워드도 위치 불일치). 개별 반응적 대응은 시간 낭비.

## 감사 결과 (2026-07-19, `roms` 원본 vs `adventure_kr.json`)

| 지표 | 값 |
|------|-----|
| 번역 cmd 0x21 런 | 1686 |
| **길이 불일치(밀림) 런** | **1490 (88%)** ← 버그 원천 |
| 한글이 원본보다 **긴** 런(패딩 불가) | **447 (27%)** ← 축약 재번역 대상 |
| 한글이 짧거나 같은 런(패딩 가능) | ~1239 |

delta 분포: 대부분 −1~−4(한글 짧음)이나 +1~+5(한글 김)도 상당수.

## 근본 예방 = **위치보존 번역** (사용자 확정 방침)

1. **한글이 짧은 런** → **원본 바이트 길이로 자동 패딩**. 공백 글리프(전각공백 idx=1→`0x11`,
   반각공백 idx=0→`0x10`, **둘 다 1바이트**라 어떤 delta든 정확히 채움)를 **런 말미**(마지막 제어코드
   뒤, `0x00` 앞)에 삽입 → 위치 100% 보존 → VM 로직 안 깨짐.
   - ⚠️ **표시 안전성 미검증**: 말미 공백이 대화상자에서 안 보이는지 **0xB0으로 프로토타입 실기 확인 필수**.
     (말미 blank는 좌측정렬 대화상자에서 대개 안 보이나, {clear} 뒤 처리 등 실측 필요.)
2. **한글이 긴 런** → **의미 보존하며 원본 바이트 길이 이하로 축약 재번역**. 축약 전까지 그 씬은
   원본유지(안전). → 이 447런이 "재번역 목록"(추려서 처리).
3. **씬 단위 규칙**: 씬의 모든 번역런이 원본 이하(패딩 가능)면 → 번역+패딩(안전). 하나라도 긴 런이면
   → 원본유지 + 축약목록에 등재. 축약 후 재빌드하면 그 씬도 안전 번역됨.
4. **cmd 0x20 컨테이너 씬**: 헤더(cmd+operand 3B)를 그대로 두고 본문만 같은 길이로 치환한다.

→ 결과: **밀린 위치가 ROM에 절대 안 실림 → 프리즈/리셋 원천 차단 → 안정 ROM으로 QA/디버그 가능.**

## 구현 계획 (build_adv.py)

- `encode_text(kr)` 결과가 원본 텍스트 길이보다 **짧으면** 말미에 공백글리프로 패딩해 정확히 일치시킴.
  cmd 0x21 런: `out += buf[prev:at+1]` + `encode_text(kr)` + **패딩** + `b"\x00"`, 총 길이 = 원본 런 길이.
- **긴 런 탐지**: `len(encode_text(kr)) > 원본텍스트길이`면 그 씬을 `LONGER_REVERT`(원본유지) + 리포트.
- 빌드 산출: **축약 필요 런 목록**(scene, at, 원본len, 한글len, 초과바이트, text_kr) → `out/retranslate_longer.json`.
- 검증: 각 씬 build 후 walk 구조 동일 확인(현행) + **Mesen 실기**(round-trip≠VM유효).

### 0xB0 프로토타입 (검증 먼저)
0xB0은 23런 중 **긴 런 3개**(@0x020A +4, @0x0369 +3, @0x0443 +5). 이 3런만 축약하면 나머지는 패딩 가능.
→ 3런 축약 후 전체 패딩 → 실기로 **프리즈 소멸 + 번역 유지** 확인 → 방식 검증 완료 → 전 씬 적용.

## ✅ 구현 완료 (2026-07-19) — 프로토타입 빌드·정적 검증 통과, 실기 대기

**build_adv.py 위치보존 구현**(커밋 예정):
- `PAD_GLYPH=0x10`(반각 공백, 1B 블랭크) / `POSPRES={0xB0}`(프로토타입: 0xB0만 위치보존, 나머지 현행 경로).
- cmd0x21 런: `enc + PAD_GLYPH*(원본텍스트바이트 − len(enc)) + 0x00` → 런 총길이 = 원본 정확 일치.
- **긴 런(패딩 불가) 탐지 → `out/retranslate_longer.json`**(전 씬 수집, Codex 축약 SSOT). pospres 씬은 긴 런 있으면 원본유지.
- 역검증: 재-walk 시 패딩 공백이 렌더에 붙으므로 기대값 = `kr + ' '*pad` 로 보정.

**0xB0 3런 축약**(adventure_kr.json):
| at | 원본 | 축약 후(byte) |
|----|------|------|
| 0x020A | 카이「아직도 이런 짓을/계속할 셈인가요！？ | 카이「아직도 이런 짓을/**할** 셈인가요！？ (27B, 「계속」생략) |
| 0x0369 | 라「곧 알게 된다 | 라「곧 **안다** (9B) |
| 0x0443 | 카이「그의 눈은…/옛날의 저와 같은 눈이었어요… | 카이「**그** 눈은…/옛날의 저와 **같았어요**… (30B) |

**정적 검증(통과)**: 전체 빌드 round-trip 214/214·렌더 214/214. 0xB0 스크립트 길이 원본과 동일(1140==1140),
23런 전부 (at,cmd)·런 경계 동일 → **VM 바이트코드 위치 100% 보존 증명**. ROM=`out/wgp2_kr.smc`(CRC32 `FE0CD6EA`).

## ✅ 실기 1차 결과 + IDA 역공학 규명 + 패딩 배치 수정 (2026-07-20)

**실기 1차(사용자)**: 0xB0 **프리즈는 소멸**(위치보존이 프리즈 원인=위치밀림을 제거). 단 **이름칸이 뒤로 밀리고**
"트라이대거는 따돌렸어요"가 맥락에 안 맞게 보이는 **표시 이상** 발생.

**IDA 역공학(idalib, docs/16)으로 원인 확정**:
- cmd0x54(조건분기) `$C0:56A0` + 평가기 `$C0:5AC6` 디스어셈블 → cmd0x54는 `op(1B)`+조건성립시 `word(2B)`를
  WRAM 슬롯($7E:0014+flag, flag=`$7695[op]`)에 등록, cmd0x56이 참조. **word는 스크립트 오프셋**.
- **위치보존은 VM 명령 스트림(cmd0x54/0x56·word offset)을 바이트 동일 보존** → 조건분기는 원본과 100% 동일 동작.
  ⇒ "이상 대화"는 **조건버그가 아니라** 유일한 변경점(텍스트런 내부=패딩)의 표시 아티팩트로 확정.
- 밀림 기전: 패딩을 `{clear}` **뒤**(0x00 앞)에 넣어, {clear}가 박스를 지운 뒤 공백이 찍히며 **펜이 전진** →
  다음 메시지 이름이 그만큼 밀림.

**수정(build_adv.py `pad_kr`)**: 패딩을 **말미 종료 제어코드 앞**에 삽입.
`...{wait}{clear}` → `...{wait}[공백]{clear}` (찍힌 즉시 지워져 비표시 + {clear} 후 펜 전진 0 → 밀림 해소).
마커 없이 글리프로 끝나는 런은 말미 트레일링 공백(좌측정렬 비표시). 정적검증: round-trip 214/214·렌더 214/214,
0xB0 스크립트 길이·런 위치 원본과 동일 유지. ROM=`out/wgp2_kr.smc`(CRC32 `01FCC0D1`).

**실기 2차(사용자)**: 0xB0 **이름칸 밀림 해소 확인**(패딩 배치 수정 검증됨). 단 다른 씬(0xC7 토우키치)에서
**새 크래시** 발견 → 위치 밀림發 크래시가 0xB0 국한이 아님을 재확인.

## ✅ 전 씬 확대 + 런 단위 원본유지 (2026-07-20)

0xB0 검증 완료(프리즈 소멸·밀림 해소) + 0xC7 신규 크래시 → **`POSPRES=None`(전 씬 위치보존)** 전환으로
위치 밀림發 크래시 **원천 차단**.

**⚠️ 씬 단위 revert의 함정 → 런 단위 원본유지로 재설계**: "긴 런 1개면 씬 통째 원본유지"(구 규칙 3)는
너무 공격적 — 20런 중 1런만 길어도 20런 전부 일본어 → 전 씬 확대 시 **1681→170 메시지로 90% 급감**.
→ **런 단위 원본유지**: 긴 런은 **그 런만** 원본 바이트(일본어, 정확히 원본 길이)로 두고 나머지 런은 번역+패딩.
모든 런이 원본 길이라 **씬 전체 위치보존 유지(크래시 0)** + 긴 런만 일본어.

**결과(build_adv, POSPRES=None)**: 번역 201씬 / **1238 메시지**, round-trip·렌더 201/201, 긴 런 443개만 일본어
(런단위 원본유지), 원본유지 31씬(cmd0x20 16 + desync 2 + 전런이긴런 13). ROM `out/wgp2_kr.smc`(CRC `0B0D8C8B`).
0xC7 위치보존 확인(len 906==906) → 크래시 소멸.

**후속 완료**:
1. 443 긴 런 축약 완료. `shortening_ledger.json`에 축약 전후를 영구 보존하고 긴 런 0 달성.
2. cmd0x20 중첩 cmd0x21 메뉴/선택지 반영 완료.
3. 2026-07-22 전수 감사에서 남아 있던 직접 cmd0x20 본문 9개를 복구. 헤더 3B와 전 런 주소를
   원본 그대로 보존해 어드벤처 **231씬·1,724메시지 전량** round-trip·역렌더 PASS.
