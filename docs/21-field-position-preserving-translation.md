# 21 · 필드/NPC 위치보존 번역·재삽입 설계

> 기존 어드벤처 번역의 위치 밀림→프리즈, 잘못된 패딩 위치→다음 이름칸 밀림 시행착오를
> 필드 레코드에서 반복하지 않기 위한 선행 설계다.

## 고정 불변식

1. 원본 ROM의 `(record src, run at, cmd, raw, orig_len)`을 번역 식별자로 고정한다.
2. `text_kr_full`은 축약 전 완역본이며 한 번 작성한 뒤 덮어쓰지 않는다.
3. `text_kr`은 실제 삽입문이다. 처음에는 full과 같고, 원본 바이트 상한을 넘는 런만 축약한다.
4. 축약은 별도 원장에 `before_kr→after_kr`, 원본 상한, 축약 이유를 남긴다.
5. 각 텍스트런의 **시작 주소와 종료 주소를 원본과 동일하게 유지**한다. 따라서 레코드 내부의
   조건분기·상대 오프셋·cmd0x20 스킵 대상은 다시 계산하지 않는다.

## 런 재작성

`field_text.json`의 `raw/orig_len`은 명령 바이트를 제외한 텍스트 바이트와 종료자 `0x00`을 포함한다.
삽입 시 허용되는 실제 텍스트 바이트는 `orig_len-1`이다.

```text
원본: [cmd][원문 바이트................][00]
한글: [cmd][text_kr][공백 패딩..........][00]
      └──────── 원본과 동일한 길이 ────────┘
```

- 한글이 짧으면 반각공백 글리프 `0x10`으로 정확히 패딩한다.
- 패딩은 문자열 맨 끝이 아니라 **마지막 가시문자 뒤, 말미 `{wait}/{clear}/개행` 블록 앞**에 둔다.
  어드벤처 실기에서 `{clear}` 뒤 패딩은 다음 이름칸을 밀었고, 개행 뒤 패딩은 다음 줄 들여쓰기를
  만들었으므로 금지한다.
- 한글이 길면 해당 런은 원본 일본어를 유지하고 축약 대기 원장에 넣는다. full 번역은 유지한다.
- 유일한 cmd0x20 런 `$C4:9B72+0x170`도 operand를 건드리지 않고 본문을 정확히 16B로 유지한다.

## 압축 레코드 배치

디컴프레스 길이가 같아도 압축 길이는 달라질 수 있으므로 두 단계로 처리한다.

1. 재압축 결과가 원본 `comp_len` 이하면 원래 슬롯에 쓴다. 남은 뒤쪽 바이트는 읽히지 않는다.
2. 원본 슬롯을 넘는 레코드만 자유공간으로 재배치하고, 카탈로그의 C2 3바이트 포인터를 전부 패치한다.
3. 재배치된 레코드의 원본 슬롯은 덮어쓰지 않는다. 누락 참조가 있더라도 일본어 원본으로 폴백한다.
4. `build_adv.py`가 먼저 사용한 자유공간을 manifest로 내보내고 필드 빌더가 그 다음 위치부터 쓴다.
   두 빌더가 같은 `FREE_POOL`을 독립적으로 0부터 할당하는 방식은 금지한다.

현재 텍스트 포함 레코드는 685개, 원본 압축 합계 67,528B다. 어드벤처 뒤 전역 자유공간 잔여는
42,579B이므로 전량 재배치 전제는 성립하지 않는다. **in-place 우선 + 초과분만 재배치**가 필수다.

## 글리프 할당

- 기존 정적 681·어드벤처 1,715·월드맵 퀴즈의 `char2idx`를 먼저 확정한다.
- 필드 신규 음절은 그 뒤에만 append한다. 기존 인덱스 변경은 0개여야 한다.
- 삽입 대상 `text_kr`만 글리프 수요에 포함한다. 보관용 `text_kr_full`의 전용 음절은 축약본이 쓰지
  않으면 ROM 폰트에 넣지 않는다.
- 글리프 부족은 빌드 오류다. 번역 단어 선택으로 억지 회피하기 전에 실제 신규 음절 수요를 보고한다.

## 번역·축약 게이트

- glossary의 인명·머신·팀·화자 말투를 준수한다.
- `{wait}/{clear}/{cN}`의 개수와 순서를 원문과 동일하게 유지한다.
- 원문보다 개행 수를 늘리지 않고 각 줄 폭은 16 이하로 한다.
- 반각 `!/?/~`를 금지하고 전각 `！/？/〜`를 쓴다.
- 축약 전후 모두 의미·화자·선택지 의미를 보존한다.

## 빌드 검증 게이트

1. 원본 디컴프레스 버퍼와 재작성 버퍼의 길이가 동일하다.
2. 모든 run의 `(at, cmd, end)`가 원본과 동일하다.
3. 텍스트 외 VM 바이트가 원본과 동일하다.
4. 재압축→재해제 결과가 재작성 버퍼와 동일하다.
5. in-place 레코드는 압축 길이≤원본 슬롯, 재배치 레코드는 C2 포인터 역검증 전수 통과.
6. 기존 어드벤처 228씬/1,715메시지, 월드맵 350문자열, 정적 681 round-trip에 회귀가 없다.
7. 마지막으로 사용자 실기에서 연구소·학교·미쿠니가·각 팀 숙소 등 대표 장소를 확인한다.
