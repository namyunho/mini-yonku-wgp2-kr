# 21 · 필드/NPC 위치보존 번역·재삽입 설계

> 기존 어드벤처 번역의 위치 밀림→프리즈, 잘못된 패딩 위치→다음 이름칸 밀림 시행착오를
> 필드 레코드에서 반복하지 않기 위한 선행 설계다.

## 진행 상태 (2026-07-22)

- `assets/translations/field_kr.json`의 고유 원문 **1,340/1,340 번역 완료**.
- 발생 텍스트런 1,411개 전부가 고유 원문 원장을 참조하며, `text_kr_full`과 `text_kr`을 분리 보존한다.
- 삽입 용량에 맞춘 축약본은 **280개**이며, `field_shortening_ledger.json`에 full→삽입본과 사유를 보존한다. 제어마커 순서·원문 이하 줄 수·줄폭 16 이하·반각 문장부호 금지 게이트를 전량 통과했다.
- `prepare_field_translation.py` 재생성 후에도 1,340개 번역 보존, 위치 입력 감사도 통과했다.
- `scripts/build_field.py` 구현 및 `build_all.py` 연결 완료. 통합 ROM은 원본과 같은 헤더리스 2MB와 ROM 크기 헤더를 유지하고 체크섬만 갱신한다.
- **실기 검증**: 월드맵 타일·스프라이트·소형 메뉴·필드 NPC 정상 확인. C2 패턴 오탐 10개를
  제외한 빌드에서 타일/OBJ 깨짐이 복구됐다.

## 고정 불변식

1. 원본 ROM의 `(record src, run at, cmd, raw, orig_len)`을 번역 식별자로 고정한다.
2. `text_kr_full`은 축약 전 완역본이며 한 번 작성한 뒤 덮어쓰지 않는다.
3. `text_kr`은 실제 삽입문이다. 처음에는 full과 같고, 원본 바이트 상한을 넘는 런만 축약한다.
4. 축약은 별도 원장에 `before_kr→after_kr`, 원본 상한, 축약 이유를 남긴다.
5. 각 텍스트런의 **시작 주소와 종료 주소를 원본과 동일하게 유지**한다. 따라서 레코드 내부의
   조건분기·상대 오프셋·cmd0x20 스킵 대상은 다시 계산하지 않는다.

## 런 재작성

`field_text.json`의 `raw/orig_len`은 명령 바이트를 제외한 텍스트 바이트와 종료자 `0x00`을 포함한다.
삽입 시 허용되는 실제 텍스트 바이트는 `orig_len-1`이다.

```text
원본: [cmd][원문 바이트................][00]
한글: [cmd][text_kr][공백 패딩..........][00]
      └──────── 원본과 동일한 길이 ────────┘
```

- 한글이 짧으면 반각공백 글리프 `0x10`으로 정확히 패딩한다.
- 패딩은 문자열 맨 끝이 아니라 **마지막 가시문자 뒤, 말미 `{wait}/{clear}/개행` 블록 앞**에 둔다.
  어드벤처 실기에서 `{clear}` 뒤 패딩은 다음 이름칸을 밀었고, 개행 뒤 패딩은 다음 줄 들여쓰기를
  만들었으므로 금지한다.
- 한글이 길면 해당 런은 원본 일본어를 유지하고 축약 대기 원장에 넣는다. full 번역은 유지한다.
- 유일한 cmd0x20 런 `$C4:9B72+0x170`도 operand를 건드리지 않고 본문을 정확히 16B로 유지한다.

## 압축 레코드 배치

디컴프레스 길이가 같아도 압축 길이는 달라질 수 있으므로 두 단계로 처리한다.

1. 재압축 크기와 관계없이 모든 레코드를 재배치하고, 카탈로그의 C2 3바이트 포인터를 전부 패치한다.
2. 필드 원본 슬롯은 하나도 덮어쓰지 않는다. C2 외 숨은 참조가 있더라도 원본으로 안전하게
   폴백하도록 유지한다. 28개만 제자리 수정한 빌드에서도 타일 깨짐이 남았다는 실기 결과에 따라
   `in-place` 최적화는 폐기했다.
3. `build_adv.py`가 중앙 씬표를 새 주소로 패치한 뒤 더는 참조되지 않는 어드벤처 원본 씬 슬롯을
   manifest로 내보내고, 필드 빌더는 이 검증된 슬롯만 best-fit decreasing으로 사용한다.
   추정 자유공간이나 필드 원본 슬롯을 재사용하는 방식은 금지한다.

텍스트 포함 레코드는 685개, 원본 압축 합계 67,528B다. 전 레코드를 재배치하고 C2 포인터를
전수 패치한다. C2 전역 패턴 검색에서 판명된 타일맵·일반 데이터 오탐 10곳은 건드리지 않는다.
중앙 씬표가 새 주소를 가리키는 어드벤처 원본 씬 슬롯 78,434B 안에 한글 압축본
77,484B 전량이 들어가며 950B가 남는다. 필드 원본 슬롯과 2MB 경계 밖 주소는 할당 후보에 넣지 않는다.

## 글리프 할당

- 기존 정적 681·어드벤처 1,724·월드맵 퀴즈의 `char2idx`를 먼저 확정한다.
- 필드 신규 음절은 그 뒤에만 append한다. 기존 인덱스 변경은 0개여야 한다.
- 삽입 대상 `text_kr`만 글리프 수요에 포함한다. 보관용 `text_kr_full`의 전용 음절은 축약본이 쓰지
  않으면 ROM 폰트에 넣지 않는다.
- 글리프 부족은 빌드 오류다. 번역 단어 선택으로 억지 회피하기 전에 실제 신규 음절 수요를 보고한다.

## 번역·축약 게이트

- glossary의 인명·머신·팀·화자 말투를 준수한다.
- `{wait}/{clear}/{cN}`의 개수와 순서를 원문과 동일하게 유지한다.
- 원문보다 개행 수를 늘리지 않고 각 줄 폭은 16 이하로 한다.
- 반각 `!/?/~`를 금지하고 전각 `！/？/〜`를 쓴다.
- 축약 전후 모두 의미·화자·선택지 의미를 보존한다.

## 빌드 검증 게이트

1. 원본 디컴프레스 버퍼와 재작성 버퍼의 길이가 동일하다.
2. 모든 run의 `(at, cmd, end)`가 원본과 동일하다.
3. 텍스트 외 VM 바이트가 원본과 동일하다.
4. 재압축→재해제 결과가 재작성 버퍼와 동일하다.
5. 전 레코드의 C2 포인터 역검증을 통과하고, 최종 메뉴·SJIS 병합 뒤에도 재배치 데이터 해시와
   필드 원본 슬롯의 원본 동일성을 다시 검사한다.
6. 기존 어드벤처 231씬/1,724메시지, 월드맵 350문자열, 정적 681 round-trip에 회귀가 없다.
7. 마지막으로 사용자 실기에서 연구소·학교·미쿠니가·각 팀 숙소 등 대표 장소를 확인한다.
