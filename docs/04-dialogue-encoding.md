# 04 · 대사 인코딩 역공학 (Dialogue Encoding) — ✅ 모델 확정·엔드투엔드 검증

본문 대사(레이스/스토리)의 인코딩을 **정적 디스어셈블 + 동적 트레이스 + 화면 실측**으로
완전 역공학하고, 실제 대사 「栄光のゴールまであとわずかだ！」로 엔드투엔드 검증했다.
메뉴/인명(plain SJIS, 02-text-survey)과는 **별개 시스템**이다.

## 결론 (핵심)

| 항목 | 값 |
|------|-----|
| 인코딩 단위 | **1바이트 가변길이** (글리프는 1~2바이트, 제어코드는 1~2바이트) |
| 저장 위치 | 스크립트 = 뱅크 **$C7** (텍스트 블록 ~`$C7:89A0`+). 폰트 = 뱅크 $CA(03-font-survey) |
| 파서 | **`$C1:9554`** (1회 호출 = 1글리프/1제어코드 처리, 반환 1=계속·0=종료) |
| 호출자 | **`$C3:7899`** `JSL $C19554`. DBR=$C7, **Y=메시지 포인터**(WRAM `$7E:C58B,X`에서 로드) |
| 글리프 인덱스 | `$26`. 폰트 오프셋 = base03(`$26`) = `16*((n&~7)*2 + (n&7))` (03-font-survey와 동일) |
| 종료자 | **`0x00`** |
| 압축 | 없음 |

→ **인코딩 모델 확정.** 다음은 추출 단계: ROM 원본 메시지 포인터 테이블 발굴( `$7E:C58B` 기록자 추적 ) + 전 글리프 인덱스→문자 표(폰트 시트 전사) + $C7 전 텍스트 영역 열거.

## 인코딩 모델 (뱅크 $C7 바이트 스트림)

파서 `$C1:9554`의 분기 로직(디스어셈블 확정):

```
byte = script[Y + counter]        ; DBR=$C7, counter=$7EA5E0(메시지 내 바이트오프셋)
if byte == 0x00:            → 종료(return 0)
elif byte <  0x10:         → 제어코드 점프테이블 JMP ($C1:960B + (byte-1)*2)
else (byte >= 0x10):       → 글리프. glyph_index = byte - 0x10
```

| 바이트 | 의미 | 인자 | 표기 |
|--------|------|------|------|
| `0x00` | 메시지 종료 | — | `{end}` |
| `0x01`–`0x04` | **2바이트 글리프 프리픽스** | 다음 1바이트 | (글리프) |
| `0x05` | 개행(줄이송) | — | `{nl}` |
| `0x06` | 대기/무동작(의미 미확정, 글리프 없음) | — | `{wait}` |
| `0x07` | 렌더 파라미터 설정(→`$7E:9A64`, VWF 변형 선택) | 1바이트 | `{param:NN}` |
| `0x08`–`0x0F` | **미정의**(점프테이블 대상이 쓰레기값) — 실스크립트 미출현 추정 | — | `{op:XX}` |
| `0x10`–`0xFF` | 단일바이트 글리프 | — | (글리프) |

### 글리프 인코딩 (가변길이·빅엔디언)

글리프 인덱스 `g`(0-베이스, 폰트 시트 순서)는 값 `(g + 0x10)`을 **빅엔디언 최소길이**로 저장한다:

- `g` in `0x000..0x0EF` → **1바이트**: `g + 0x10` (범위 `0x10..0xFF`, 항상 ≥0x10이라 제어코드와 안 겹침)
- `g` in `0x0F0..0x4EF` → **2바이트**: `prefix, lo` where `prefix = (g+0x10)>>8` (=`0x01..0x04`), `lo = (g+0x10)&0xFF`

즉 제어코드로 보이는 `0x01`–`0x04`는 사실 **16비트 글리프 인덱스의 상위바이트**(1~4)다. 파서 핸들러 `$C1:95B8`(코드 0x01–0x04 공용)이 `XBA`로 상위바이트에 코드값을 넣고 다음 바이트를 하위에 실어 `glyph=(code<<8|next)-0x10`을 만든다. 글리프 공간 `0x000..~0x4EF`(≈1264)이 ~1056 글리프(03-font-survey)를 커버.

### 제어코드 핸들러 요지 (디스어셈블)

- **`0x05` 개행** (`$C1:95D0`): X-커서(`$7EA5E2`)를 좌마진(`$000A32`)으로 리셋, Y-커서(`$7EA5E4`) += `$0300`, `$0C01` 이상이면 0으로 wrap.
- **`0x06`** (`$C1:95F2`): `BRA $95A8` — 글리프 없이 return 1(계속). 바이트 1개 소비, 화면 효과 미확정.
- **`0x07` 파라미터** (`$C1:95F4`): 다음 1바이트를 읽어 `<<1` 후 `$7E:9A64`에 저장. 이 값은 글리프 페치(`$C0:6881 LDX $9A64; JSR ($6A64,X)`)의 2차 VWF 점프 인덱스 → 렌더 변형(폰트 스타일/크기/색 모드 추정) 선택. 1바이트 인자.

## 엔드투엔드 검증 (frame 1200 레이스 대사)

화면 대사 **「栄光のゴールまであとわずかだ！」(15자)** ↔ 스크립트 `$C7:89FF` (동적 트레이스로 위치 확정):

```
raw: 02 FD | 01 6C | 38 AE C5 98 3E 64 20 33 4B 5E 25 61 E4 | 00
```

|---|------|--------|-----------|------|
| 1 | 栄 | `02 FD` | `0x2ED` | 2바이트(한자) |
| 2 | 光 | `01 6C` | `0x15C` | 2바이트(한자) |
| 3 | の | `38` | `0x028` | |

| 5 | ー | `C5` | `0x0B5` | 장음부 |

| 7 | ま | `3E` | `0x02E` | |

| 10 | と | `33` | `0x023` | |

15 글리프 = 15 문자 정확 일치. **모델 완전 검증.**

### 글리프 시트 배열 (확정된 앵커로 역산)

폰트 시트($CA:1137) 순서 = 숫자 0-9 → 히라가나 → 카타카나 → 라틴 → 기호 → 한자(03-font-survey)와 정합:

- 숫자: `0x000`–`0x009` (0-9)
- 히라가나 청음 오십음도: **`あ=0x010`** 부터 연속. か=0x015, さ행=0x01A~, の=0x028, と=0x023, ま=0x02E, わ=0x03B … (오십음도 순 확정)
- 히라가나 탁·반탁음: 청음 블록 뒤. ず=0x04E, だ=0x051, で=0x054 (が행부터 +3 간격 정합)
- 카타카나: ル=0x088, ゴ=0x09E, 장음부 ー=0x0B5
- 기호: 전각 ！=0x0D4
- 한자: 光=0x15C, 栄=0x2ED (2바이트 영역)

> 전 인덱스→문자 표(≈1056)는 추출 단계에서 폰트 시트 전사로 완성한다(위 15개는 검증된 앵커).

## 메시지 포인터 (호출자 $C3:7899 분석)

```asm
$C3:788E  LDA $7EC58B,X   ; 메시지 포인터 ← WRAM 슬롯 테이블 $7E:C58B,X (16비트)
$C3:7892  TAY             ; Y = 메시지 베이스 포인터
$C3:7894  PEA #$C700 / PLB / PLB   ; DBR = $C7
$C3:7899  JSL $C19554     ; 파서(Y=포인터, counter=$7EA5E0=바이트오프셋)
$C3:789E  TAY / BNE       ; 반환 1이면 다음 글리프 루프
```

- 메시지 포인터는 **런타임 WRAM 큐 `$7E:C58B,X`**에서 로드된다(ROM 직접 아님). 병렬 슬롯 배열 `$7E:C5CB/$C64B/$C74B`(속성/뱅크 등 추정).
- 파서의 다른 호출자: `$C1:2248/6E8B/7D18`, `$C3:7899/7F08`, `$C10:506A/516A` — 화면·컨텍스트별.
- **추출 단계 진입점**: `$7E:C58B`에 쓰는 코드를 추적하면 ROM 원본 메시지 포인터 테이블(대사 ID→오프셋)이 나온다.

## 재현 (도구)

- **디스어셈블러**: `scripts/disasm.py <rom> <file_off> <count> [--m 0|1] [--x 0|1]` — 최소 65816 선형 디스어셈블(M/X 추적). 파서 `python scripts/disasm.py <rom> 0x019554 0xE0 --m 0 --x 0`.
- **스크립트 위치 트레이스**: `scripts/lua/script_trace.lua` — 뱅크 $C7 CPU버스 읽기 후킹 → 스크립트 addr·바이트를 소비 순서로 덤프(`tmp/trace/script_trace.txt`).
- **호출자 트레이스**: `scripts/lua/parser_caller.lua` — 파서 진입 후킹, 복귀주소·Y·DBR·counter 포착(`tmp/trace/parser_caller.txt`).
- **디코더**: `scripts/decode_script.py` — 위 인코딩 모델 구현. `--tbl glyph_index<TAB>char`로 문자 표 적용. 라운드트립 인코더는 재삽입 단계에서 추가.
- Mesen2 실행: `Mesen.exe "<rom>" "<script.lua>"` (설정 `AllowIoOsAccess=true`, `ScriptTimeout` 상향 — 반영됨). Start 펄스로 타이틀→모드선택→레이스 진행 후 frame~1087에 대사 렌더.
