# 13 · 어드벤처 원본유지(번역 보류) 씬 — 재번역 추적 SSOT

인게임 QA에서 **번역이 VM을 깨뜨리는 씬**(리셋·행·프리즈)을 발견해, 플레이 가능성을 위해
`build_adv.py`가 해당 씬을 **통째 원본 유지**(번역 스킵)한다. 번역 자체는 `adventure_kr.json`에
그대로 있으므로, **build 로직을 고쳐 VM을 안 깨뜨리게 하면 재적용(재번역)** 된다.

⚠️ 현재 이 씬들은 게임에서 **원본 일본어가 $CA 한글폰트로 렌더돼 깨져 보인다**(플레이는 정상).

## 원본유지 씬 목록 (재번역 대상)

### A. cmd 0x20 특수명령 (16씬) — 커밋 f9519f6
메뉴 컨테이너·아이템 지급·레벨업 표시 등. cmd 0x20은 단순 텍스트런이 아니라 특수 명령이고,
op(길이/구조)를 재작성하면 VM이 깨진다(대화창 행·선택메뉴 미표시·씬 종료 후 리셋).

| 씬 | 내용 |
|----|------|
| 0x29 | 이벤트 아이템 WGP 티켓 |
| 0x35 | 이벤트 아이템 안테나 |
| 0x64 | 파이터 코스엔트리(선택메뉴: 코스출전/그리드변경/그만두기) |
| 0x69 | **레벨이 올랐다(레벨업, 매 레이스 후)** |
| 0x6B | 휴일이라 잠겨 있다(잠김 메시지) |
| 0x6F | 아이템 입수 |
| 0xA7 | 레츠→카이조 반지 돌려주기 |
| 0xAC·0xAD·0xB3·0xB4·0xB5·0xB6·0xB7·0xB9·0xBA | 지로마루 야에짱 아크(특훈) |

### B. desync 앵커재빌드 (2씬) — 커밋 c782122
walk가 씬 구조를 못 잡아(desync) 앵커 기반 재빌드했으나 VM 유효성 미보장 → 원본유지.

| 씬 | 내용 |
|----|------|
| 0xA8 | 카이조 반지 분실 setup |
| 0xB2 | 지로마루 코스에서 반지 발견 |

### C. 조건분기 런타임 붕괴 씬 (프리즈) — build_adv `FREEZE_REVERT`
조건 분기(cmd 0x54)의 **런타임 제어흐름이 텍스트 번역(런 밀림)으로 깨져 프리즈**. 정적으로
원인 규명 실패(원본·빌드 둘 다 23런·desync 없음·구조 동일·제어코드 일치·명확한 offset 명령 없음).

| 씬 | 내용 | 상태 |
|----|------|------|
| 0xB0 | 료·카이 대화(카이 "큰소리치던 것에 비하면 대단할 건 없었네요") | **VM-붕괴 확정**(신뢰 세이브 테스트: 0xB0만 원복→프리즈 소멸). 원본유지 |

**격리 과정(SSOT)**: no-SJIS ROM으로 SJIS khook2 배제 → 어드벤처 통째 원복=프리즈 소멸 → 창(0xAF/0xB0/0xB1)
바이섹션 → 0xB0 단독 원복=프리즈 소멸로 **0xB0 확정**. 씬 트레이서(`trace_scene_id.lua`)로 로드순서 확인.

⚠️ **프로액티브 전수탐지 불가 판정**: 조건분기(cmd 0x53/0x54) 씬이 **86개**(230 번역 중)인데 그중
0xB0만 깨진다. 정적으로 0xB0(깨짐)과 나머지 85개(정상)를 구별할 신호가 없음(offset 명령 hit비율 전부
낮음, cond 워드도 위치 불일치). → **깊은 cmd 0x54 스킵 메커니즘 RE 없이는 프로액티브 탐지 불가.**
현실적 방침: 확정 씬만 `FREEZE_REVERT`에 원본유지하고, 추가 프리즈/리셋은 **세이브+씬트레이서로 반응적**
확정 후 추가(각 ~15분). 근본 재번역은 cmd 0x54 스킵 RE 후 후속.

## 근본 재번역 방법 (후속 과제)

1. **cmd 0x20 컨테이너**: op 상위바이트(0x02/0x31) 플래그·내부 임베드 0x21 런 포맷 RE →
   컨테이너 구조·op 보존하며 내부 텍스트만 치환.
2. **조건분기 씬**: cmd 0x54/점프의 **내부 offset을 텍스트 밀림 delta만큼 보정**하도록 build_adv 확장
   (어느 오퍼랜드가 씬내 offset인지 명령별 RE 필요). 또는 런 바이트길이 보존(패딩) 방식 검토.
3. **desync 씬**: 씬 구조 정밀 재파싱 후 텍스트런만 치환.

재번역 시 각 씬 build 후 **Mesen 실기 검증 필수**(round-trip 통과 ≠ VM 유효).
