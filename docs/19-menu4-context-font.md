## 결론

월드맵 X 메뉴와 조작방법 튜토리얼은 같은 압축 자원 `$D9:0000`의 앞 4KB를
8×8 2bpp 소형 글꼴 페이지로 사용한다. 용어집과 지도도 같은 원본 자원을
각각 독립된 로더로 읽는다. 원본 자원을 세 문맥 전용 한글 글꼴로 재압축하고,
각 화면이 원래 자원을 읽던 포인터만 새 자원으로 돌린다.

- 실행 코드 훅 없음
- NMI·WRAM 활성 플래그 없음
- 추가 DMA와 프레임별 재주입 없음
- 기존 렌더러와 원본 화면 전환 순서를 그대로 유지
- 모든 라벨은 1음절 1바이트 직접 타일 인덱스
- 고정 슬롯은 in-place, 포인터 테이블 문자열은 원래 블록 안에서 재패킹

이 방식으로 기존 구현에서 화면 이동·튜토리얼 페이지 전환 때 발생하던 보라색
노이즈가 사라졌다.

## 원본에서 독립 확인한 데이터 경로

### 공통 글꼴 자원

`$D9:0000`은 2바이트 해제 길이 헤더와 LZSS 스트림으로 구성된다.

| 항목 | 값 |
|---|---|
| 해제 크기 | 5,984B (`0x1760`) |
| 원본 압축 크기 | 3,768B(헤더 포함) |
| 전체 해제 SHA-256 | `78dfbb47aff74d3d6ccfa055ab0cb5975cdf779fcc617edb9db2ebf266bb43dd` |
| 앞 4KB 글꼴 SHA-256 | `9f84cb8c101db514d4c4f6218099d1a90b95daea24d4d1c1fc2ce340986bd155` |

Mesen VRAM 덤프에서 월드맵과 튜토리얼의 소형 글꼴 페이지는 이 앞 4KB와
바이트 단위로 일치했다.

### 월드맵

`$C0:94D4`가 `$D9:0000`을 `$C0:0D52`에 전달한다. 해제 결과 `$7F:1000`의
4KB를 VRAM word `$4800`으로 DMA하며, 월드맵 BG3 문자 베이스 word `$4000`에
타일 페이지 `$100`을 더한 위치다.

직접 타일 라벨 원본 위치:

| 화면 | PC 주소 | 원문 |
|---|---:|---|
| X 메뉴 | `0x0395BE` | セッティング |
| X 메뉴 | `0x0395CA` | グリッドへんこう |
| X 메뉴 | `0x0395DA` | アイテム |
| 아이템 서브메뉴 | `0x039201` | マップ |
| 아이템 서브메뉴 | `0x039208` | そうさマニュアル |
| 아이템 서브메뉴 | `0x039212` | ようごしゅう |
| 세팅 서브메뉴 | `0x0399C0` | イージー |
| 세팅 서브메뉴 | `0x0399CA` | マニュアル |

렌더러는 원본 `$C0:1B4B` 그대로다. `0xFF`는 빈 타일이며, 별도 한글 마커를
추가하지 않는다.

탁점 문자는 `00 94/95 + 본문 타일`의 3바이트지만 화면에서는 한 칸이다.
따라서 원본 바이트 수를 표시 폭으로 간주해 `FF`를 채우면 우측 테두리까지
덮어쓴다. 빌더는 각 행에서 탁점 오버레이 두 바이트를 제외한 실제 셀 수를
계산하고, 멀티라인 프로그램 전체를 다시 조립해 원래 행 이동(`00 02`)과
종료(`00 00`) 제어를 보존한다. 종료 뒤 고정 슬롯 잔여만 `FF`로 채운다.

### 용어집

용어집 진입 로더 `$C3:9DC5`는 원본 `$D9:0000`을 해제해 VRAM word
`$4800`으로 전송한다. 분류 포인터는 `$C3:A142`, 항목 포인터는
`$C3:A14F~A1C4`, 문자열 블록은 `$C3:A1C5~A39A`다.

분류 4개와 항목 38개를 번역한다. 인물군에는 요청된 10명 뒤에 실제 ROM
테이블에만 이어지는 4명(미쿠니 치이코, 미즈사와 히코사, 미니욘 파이터,
야나기 타마미)이 있어 같은 문맥 폰트에서 원문이 깨지지 않도록 함께 처리한다.
번역 문자열은 340B로 원래 470B 블록 안에 다시 채우고 42개 포인터를 갱신한다.
하단 상세 설명은 기존 어드벤처 대사 엔진을 사용하므로 이 변경과 독립적이다.

### 지도

지도 진입 로더 `$C3:A835`도 원본 `$D9:0000`을 별도로 읽는다. 목적지 포인터
11개는 `$C3:AB75~AB95`, 문자열 블록은 `$C3:AB97~ABF8`이다. 번역문은
78B로 원래 98B 블록 안에 재패킹한다. 원본 실제 문자열 `せいばけ`는
사용자 메모의 `せいばてい`가 아니라 의미에 맞춰 `세이바 집`으로 번역했다.
짧은 `つちけんレース`는 화면 폭에 맞춰 `연구소 경기장`으로 표기한다.

### 조작방법 튜토리얼

튜토리얼 진입 스크립트 `$C3:67A4`가 인라인 자원 포인터 `$D9:0000`을
공통 해제기 래퍼 `$C3:53C7`에 전달한다. 해제 결과는 `$C3:5438`에서 VRAM
word `$3800`으로 4KB DMA된다. 튜토리얼 BG3 문자 베이스 word `$3000`과
타일 페이지 `$100`의 조합이다.

라벨 데이터는 `$C7:B183~B466`에 있으며 각 항목은 다음 두 레코드다.

```text
[상단 길이][탁점·반탁점 타일열]
[하단 길이][본문 타일열]
```

소비 루틴은 `$C3:7B6C/$C3:7B73`(상단)과 `$C3:7B8A/$C3:7B91`(하단)이다.
한글 레코드는 상단을 0으로 비우고 하단에 1바이트 한글 타일을 기록한다.
영문 약어 `SP/CN/PW/DF/WT/DP/BP/WGP`와 공백은 원본 타일을 보존한다.

## 한글 타일 설계

월드맵·튜토리얼 문맥은 72음절이다. 원본 라벨이 사용하던 가나 타일과
빈 타일 `FC/FD/FE`를 재사용한다.
용어집은 92음절, 지도는 44음절이며 각 화면에서 사라지는 원문 가나·탁점·
가운데점 타일을 독립적으로 재사용한다. `FIMA/WGP/GP/TRF/NA/CCP/XTO/J/4`에
필요한 라틴·숫자 타일과 공백·종단 타일은 보호한다.

한글 원본은 `8pt_font/font-007242d37349daf3.bin`이며 1bpp 글리프를 SNES
2bpp 16바이트 타일로 변환한다. 원본 `$D9` 바닥선에 맞춰 1px 아래로 정렬한다.

긴 번역은 화면 폭과 문맥별 음절 제한에 맞춰 의미를 유지하는 짧은 표현을 사용한다.
예: `アクション→동작`, `マニュアル→수동`, `ウィンドウ→창`,
`ステアリング→조향`, `ポーズ→정지`, `イージー→쉬운조작`.

## ROM 배치와 패치 지점

월드 문맥 자원은 원본에서 전부 `0xFF`인 `$C7:E000~FFFF`에 기록한다.
용어집·지도 자원은 원본 `0xFF` 자유 영역 `$C2:D448~FFFF`에 연속 배치한다.

| 패치 | 원본 | 변경 |
|---|---|---|
| 월드맵 자원 인자 `$C0:94D4` | `PEA #$00D9; PEA #$0000` | `PEA #$00C7; PEA #$E000` |
| 튜토리얼 인라인 포인터 `$C3:67A8` | `00 00 D9` | `00 E0 C7` |
| 용어집 자원 인자 `$C3:9DC5` | `PEA #$00D9; PEA #$0000` | `PEA #$00C2; PEA #$D448` |
| 지도 자원 인자 `$C3:A835` | `PEA #$00D9; PEA #$0000` | `PEA #$00C2; PEA #$E280` |
| 튜토리얼 라벨 | `$C7:B183~B466` | 43개 실제 레코드 in-place |
| 월드맵 메뉴 라벨 | 위 PC 주소 8곳 | 고정 슬롯 in-place, 나머지 `FF` 패딩 |
| 용어집 | `$C3:A142~A39A` | 42개 포인터·문자열 블록 재패킹 |
| 지도 | `$C3:AB75~ABF8` | 11개 포인터·문자열 블록 재패킹 |

빌더는 원본 ROM 해시, 원본 포인터와 라벨 바이트, 자유 영역 `0xFF`, LZSS 왕복,
문맥별 음절 수, 모든 레코드 길이와 재디코드를 assert한다. 마지막에 SNES 헤더
체크섬과 보수를 다시 계산한다.

## 빌드와 검증

```bash
python3 scripts/build_menu4_reclean.py
python3 scripts/test_roundtrip.py
```

검증 결과:

- 출력: `out/menu4_reclean.smc`(gitignore, 비커밋)
- CRC32 `DE1D9594`, MD5 `0c50a22cbe054c65f40bacda7743a66e`
- SNES 체크섬 `2171`, 보수 `DE8E`
- LZSS 재압축→해제 왕복 PASS
- 동일 입력 결정론적 재빌드 SHA-256 일치
- 대사 681개 `raw==ROM`, `encode(parse)==raw` 전량 PASS
- Mesen2: X 메뉴, 아이템·세팅 서브메뉴, 튜토리얼 전 페이지, 화면 전환 정상
- Mesen2: 용어집 4분류·42항목과 지도 11목적지 표시 정상
- 수동 세팅 X메뉴 실험은 공유 글꼴 충돌로 제외(`docs/20` 인계)
- 기존 `wgp2_kr.srm`을 재사용할 때는 테스트 ROM basename을 `wgp2_kr`로 유지

원본 ROM과 출력 ROM은 절대 커밋하지 않는다.
