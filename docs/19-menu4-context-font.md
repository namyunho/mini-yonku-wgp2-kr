# 19 · 월드맵 메뉴·조작방법 문맥 폰트 한글화

> 2026-07-22 Mesen2 실기 검증 완료. `docs/18`의 FE 마커·VRAM 800·NMI 재DMA
> 설계를 폐기하고, 원본의 자원 수명주기를 그대로 이용하는 최종 방식이다.

## 결론

월드맵 X 메뉴와 조작방법 튜토리얼은 같은 압축 자원 `$D9:0000`의 앞 4KB를
8×8 2bpp 소형 글꼴 페이지로 사용한다. 이 자원을 문맥 전용 한글 글꼴로
재압축하고, 두 화면의 자원 포인터만 새 자원 `$C7:E000`으로 돌린다.

- 실행 코드 훅 없음
- NMI·WRAM 활성 플래그 없음
- 추가 DMA와 프레임별 재주입 없음
- 기존 렌더러와 원본 화면 전환 순서를 그대로 유지
- 모든 라벨은 1음절 1바이트 직접 타일 인덱스이며 원래 슬롯 안에서 수정

이 방식으로 기존 구현에서 화면 이동·튜토리얼 페이지 전환 때 발생하던 보라색
노이즈가 사라졌다.

## 원본에서 독립 확인한 데이터 경로

### 공통 글꼴 자원

`$D9:0000`은 2바이트 해제 길이 헤더와 LZSS 스트림으로 구성된다.

| 항목 | 값 |
|---|---|
| 해제 크기 | 5,984B (`0x1760`) |
| 원본 압축 크기 | 3,768B(헤더 포함) |
| 전체 해제 SHA-256 | `78dfbb47aff74d3d6ccfa055ab0cb5975cdf779fcc617edb9db2ebf266bb43dd` |
| 앞 4KB 글꼴 SHA-256 | `9f84cb8c101db514d4c4f6218099d1a90b95daea24d4d1c1fc2ce340986bd155` |

Mesen VRAM 덤프에서 월드맵과 튜토리얼의 소형 글꼴 페이지는 이 앞 4KB와
바이트 단위로 일치했다.

### 월드맵

`$C0:94D4`가 `$D9:0000`을 `$C0:0D52`에 전달한다. 해제 결과 `$7F:1000`의
4KB를 VRAM word `$4800`으로 DMA하며, 월드맵 BG3 문자 베이스 word `$4000`에
타일 페이지 `$100`을 더한 위치다.

직접 타일 라벨 원본 위치:

| 화면 | PC 주소 | 원문 |
|---|---:|---|
| X 메뉴 | `0x0395BE` | セッティング |
| X 메뉴 | `0x0395CA` | グリッドへんこう |
| X 메뉴 | `0x0395DA` | アイテム |
| 아이템 서브메뉴 | `0x039201` | マップ |
| 아이템 서브메뉴 | `0x039208` | そうさマニュアル |
| 아이템 서브메뉴 | `0x039212` | ようごしゅう |
| 세팅 서브메뉴 | `0x0399C0` | イージー |
| 세팅 서브메뉴 | `0x0399CA` | マニュアル |

렌더러는 원본 `$C0:1B4B` 그대로다. `0xFF`는 빈 타일이며, 별도 한글 마커를
추가하지 않는다.

### 조작방법 튜토리얼

튜토리얼 진입 스크립트 `$C3:67A4`가 인라인 자원 포인터 `$D9:0000`을
공통 해제기 래퍼 `$C3:53C7`에 전달한다. 해제 결과는 `$C3:5438`에서 VRAM
word `$3800`으로 4KB DMA된다. 튜토리얼 BG3 문자 베이스 word `$3000`과
타일 페이지 `$100`의 조합이다.

라벨 데이터는 `$C7:B183~B466`에 있으며 각 항목은 다음 두 레코드다.

```text
[상단 길이][탁점·반탁점 타일열]
[하단 길이][본문 타일열]
```

소비 루틴은 `$C3:7B6C/$C3:7B73`(상단)과 `$C3:7B8A/$C3:7B91`(하단)이다.
한글 레코드는 상단을 0으로 비우고 하단에 1바이트 한글 타일을 기록한다.
영문 약어 `SP/CN/PW/DF/WT/DP/BP/WGP`와 공백은 원본 타일을 보존한다.

## 한글 타일 설계

대상 번역에 필요한 완성형은 72음절이다. 원본 라벨이 사용하던 가나 타일
69개와 빈 타일 `FC/FD/FE`를 문맥 전용 한글 타일로 재사용한다. 라틴·숫자·공백과
페이지 번호 타일은 보호한다.

한글 원본은 `8pt_font/font-007242d37349daf3.bin`이며 1bpp 글리프를 SNES
2bpp 16바이트 타일로 변환한다. 원본 `$D9` 바닥선에 맞춰 1px 아래로 정렬한다.

긴 번역은 기존 슬롯과 72음절 제한에 맞춰 의미를 유지하는 짧은 표현을 사용한다.
예: `アクション→동작`, `マニュアル→수동`, `ウィンドウ→창`,
`ステアリング→조향`, `ポーズ→정지`, `イージー→쉬운조작`.

## ROM 배치와 패치 지점

새 자원은 원본에서 전부 `0xFF`임을 확인한 `$C7:E000~FFFF`에 기록한다.
재압축 결과는 헤더 포함 3,741B다.

| 패치 | 원본 | 변경 |
|---|---|---|
| 월드맵 자원 인자 `$C0:94D4` | `PEA #$00D9; PEA #$0000` | `PEA #$00C7; PEA #$E000` |
| 튜토리얼 인라인 포인터 `$C3:67A8` | `00 00 D9` | `00 E0 C7` |
| 튜토리얼 라벨 | `$C7:B183~B466` | 43개 실제 레코드 in-place |
| 월드맵 메뉴 라벨 | 위 PC 주소 8곳 | 고정 슬롯 in-place, 나머지 `FF` 패딩 |

빌더는 원본 ROM 해시, 원본 포인터와 라벨 바이트, 자유 영역 `0xFF`, LZSS 왕복,
72음절 수, 모든 레코드 길이와 재디코드를 assert한다. 마지막에 SNES 헤더
체크섬과 보수를 다시 계산한다.

## 빌드와 검증

```bash
python3 scripts/build_menu4_reclean.py
python3 scripts/test_roundtrip.py
```

검증 결과:

- 출력: `out/menu4_reclean.smc`(gitignore, 비커밋)
- CRC32 `DDDD3AC4`, MD5 `52eacf77889e96f09e77e5b20e55d18c`
- SNES 체크섬 `C228`, 보수 `3DD7`
- LZSS 재압축→해제 왕복 PASS
- 동일 입력 결정론적 재빌드 SHA-256 일치
- 대사 681개 `raw==ROM`, `encode(parse)==raw` 전량 PASS
- Mesen2: X 메뉴, 아이템·세팅 서브메뉴, 튜토리얼 전 페이지, 화면 전환 정상
- 기존 `wgp2_kr.srm`을 재사용할 때는 테스트 ROM basename을 `wgp2_kr`로 유지

원본 ROM과 출력 ROM은 절대 커밋하지 않는다.
