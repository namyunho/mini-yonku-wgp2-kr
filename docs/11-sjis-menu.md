# 11 · SJIS 시작 메뉴 시스템 — 🟡 메커니즘 역공학 완료

시작 화면 세이브 메뉴(`はじめから`/`つづきから`/`うつす`/`けす` + FILE1~4·NO DATA)의 한글화.
파서(673)·어드벤처와 **또 다른 SJIS 텍스트+타일폰트 시스템**.

## 화면 도달
부팅→닌텐도로고→크레딧(f360)→타이틀로고(f960)→**Start→시작메뉴(f~1120)**. 트레이스 `shot_menu.lua`.

## 문자열 (표준 SJIS, 비압축, 널종료)
- 타이틀 메뉴: `$C0:71B9` = `はじめから　つづきから　うつす　　　けす` / `ほぞん…もどる…`
- (docs/02: 머신명 $C0:EB73, 인명 $C0:F4B9, UI $C1:C595)

## ✅ SJIS → 타일 변환 (렌더러 `$C1:965E`)
2바이트 SJIS 문자를 읽어 타일 인덱스로 변환 후 WRAM 타일맵버퍼 `$7E:0000`에 기록:
```
LDA [str],Y ; 0이면 끝                 ; SJIS 상위바이트 → $01
LDA [str],Y ; SBC #$40 ; ASL → $03      ; (하위-0x40)*2
X = ($01 & 0x7F)*2 ; LDA $C1D1C3,X → 행베이스
X = 행베이스 + $03 ; LDA $C1D1C3,X → 타일인덱스($05)
$05 + $07(베이스) → $7E:0000,X (타일맵)   ; 본체 타일
$06(있으면=탁점타일) + $07 → $7DFFC0,X    ; 위 칸에 ゛/゜ 오버레이
```
- **변환표 = `$C1:D1C3`** (2단계: SJIS상위→행베이스 오프셋, +하위오프셋→타일인덱스). 표값이 2바이트면 `(탁점타일<<8)|본체타일`(예 じ=0x940C: 본체 0x0C=し + 탁점 0x94).
- 메뉴 폰트 = **소형**(히라가나 등이 타일 0x00~0x30 저인덱스). 실제 VRAM은 **4bpp, 타일 ~0x400+**(베이스 $07 가산). 확인: `tmp/trace/menu/all_4bpp.png`(はじめから 등 판독됨). VRAM/CGRAM 덤프 `dump_menu_vram.lua`.

## 한글화 방법 (계획)
1. **한글 글리프 타일**을 메뉴 폰트에 추가(4bpp). 폰트 ROM 소스 위치·압축여부 확인 필요(타이틀/메뉴 그래픽은 LZSS 압축일 가능성 → `lzss.py` 적용).
2. **변환표 `$C1:D1C3`에 한글 매핑 추가**: 미사용 SJIS 코드(또는 재활용)를 한글 타일인덱스로.
3. **메뉴 문자열 재작성**: `$C0:71B9` 등을 한글 매핑 SJIS 코드로. 한글 소요 음절 적음(처음부터/이어하기/복사/지우기/파일/…≈20~30자).
4. Mesen 확인.

## ✅ 메뉴 폰트 = 비압축 ROM $C9 (직접 편집 가능!)
- VRAM 메뉴폰트 타일을 ROM 검색 → **VRAM tile 1121 = ROM `$C9:4468`** 정확일치(비압축). 즉 메뉴 폰트는 **뱅크 $C9에 비압축 4bpp**로 존재 → **YY-CHR/직접 바이트패치로 편집 가능**(LZSS 불필요).
- 폰트 블록 대략 `$C9:38xx~49xx`(추정, VRAM tile↔ROM 선형). 렌더 `tmp/trace/menu/romfont_4bpp.png`.
- 타일↔ROM: VRAM tile V = tile_index($C1D1C3표) + 베이스 $07. ROM(V) ≈ 0x094468 + (V−1121)*32. **$07(베이스) 미확정** → 정확한 tile_index↔ROM 매핑에 필요.

## 렌더 경로 이중성 (확정 필요)
- **`$C1:965E`**: SJIS 2바이트 → 표 $C1:D1C3 → 타일. (트레이스서 $C0:71B9 읽음)
- **`$C0:1B4B`**: `[$0F]` 문자열을 **직접 타일코드**로 읽어 배치(+베이스 $01), 0xFF·저코드(0-2) 특수처리. (역시 $C0:71B9 읽음)
- 둘 다 메뉴영역을 읽어 어느 것이 타이틀 메뉴 실렌더인지 미확정 → **타일맵버퍼 $7E:0000 쓰기 트레이스로 확정 필요**.

## 한글 주입 계획 (경로 확정 후)
방식A(권장): 폰트 빈 슬롯에 한글타일 추가 + 표/문자열에 매핑 추가(기존 글리프 안 깨짐).
방식B(빠름): はじめから 등이 쓰는 타일을 한글로 덮어씀(공유타일 주의 — 다른 항목 깨질 수 있음).
- 소요 한글 적음(처음부터/이어하기/복사/지우기 등 ~20자). 폰트·표·문자열 전부 ROM $C9/$C1/$C0에 있어 직접 패치 가능.

## 미확정(주입 전 필요)
- 실렌더 경로($C1:965E vs $C0:1B4B) + 베이스($07/$01) → tile_index↔ROM 확정.
- 폰트 빈 슬롯 범위, 표 $C1:D1C3 여유(미사용 SJIS 코드).
