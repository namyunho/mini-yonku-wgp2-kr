# 11 · SJIS 시작 메뉴 시스템 — ✅ 한글 주입 구현·실기 검증 완료 (비파괴 ASM 훅; 세이브시 색변화 검증만 잔여)

시작 화면 세이브 메뉴(`はじめから`/`つづきから`/`うつす`/`けす` + FILE1~4·NO DATA)의 한글화.
파서(673)·어드벤처와 **또 다른 SJIS 텍스트+타일폰트 시스템**.

## 화면 도달
부팅→닌텐도로고→크레딧(f360)→타이틀로고(f960)→**Start→시작메뉴(f~1120)**. 트레이스 `shot_menu.lua`.

## 문자열 (표준 SJIS, 비압축, 널종료)
- 타이틀 메뉴: `$C0:71B9` = `はじめから　つづきから　うつす　　　けす` / `ほぞん…もどる…`
- (docs/02: 머신명 $C0:EB73, 인명 $C0:F4B9, UI $C1:C595)

## ✅ SJIS → 타일 변환 (렌더러 `$C1:965E`)
2바이트 SJIS 문자를 읽어 타일 인덱스로 변환 후 WRAM 타일맵버퍼 `$7E:0000`에 기록:
```
LDA [str],Y ; 0이면 끝                 ; SJIS 상위바이트 → $01
LDA [str],Y ; SBC #$40 ; ASL → $03      ; (하위-0x40)*2
X = ($01 & 0x7F)*2 ; LDA $C1D1C3,X → 행베이스
X = 행베이스 + $03 ; LDA $C1D1C3,X → 타일인덱스($05)
$05 + $07(베이스) → $7E:0000,X (타일맵)   ; 본체 타일
$06(있으면=탁점타일) + $07 → $7DFFC0,X    ; 위 칸에 ゛/゜ 오버레이
```
- **변환표 = `$C1:D1C3`** (2단계: SJIS상위→행베이스 오프셋, +하위오프셋→타일인덱스). 표값이 2바이트면 `(탁점타일<<8)|본체타일`(예 じ=0x940C: 본체 0x0C=し + 탁점 0x94).
- 메뉴 폰트 = **소형**(히라가나 등이 타일 0x00~0x30 저인덱스). 실제 VRAM은 **4bpp, 타일 ~0x400+**(베이스 $07 가산). 확인: `tmp/trace/menu/all_4bpp.png`(はじめから 등 판독됨). VRAM/CGRAM 덤프 `dump_menu_vram.lua`.

## 한글화 방법 (계획)
1. **한글 글리프 타일**을 메뉴 폰트에 추가(4bpp). 폰트 ROM 소스 위치·압축여부 확인 필요(타이틀/메뉴 그래픽은 LZSS 압축일 가능성 → `lzss.py` 적용).
2. **변환표 `$C1:D1C3`에 한글 매핑 추가**: 미사용 SJIS 코드(또는 재활용)를 한글 타일인덱스로.
3. **메뉴 문자열 재작성**: `$C0:71B9` 등을 한글 매핑 SJIS 코드로. 한글 소요 음절 적음(처음부터/이어하기/복사/지우기/파일/…≈20~30자).
4. Mesen 확인.

## ✅ 베이스 $07·렌더러·폰트 소스 전부 확정 (2026-07-14)

### 렌더러 $C1:965E 완전 디스어셈블 (파일 0x01965E)
프롤로그 `PHD; PHA×4; TSC; TCD`로 DP를 스택프레임에 재매핑 → 첫 PHA가 진입 A를 밀어 **`$07`(+$08) = 진입 시 A 레지스터**가 됨(루프 내내 불변). 즉 **베이스 $07은 호출자가 A로 전달**. 루프: `tile = (표값 & 0xFF) + $07` → `$7E:0000,X`(타일맵버퍼), 탁점타일 `(표값>>8) + $07` → `$7D:FFC0,X`(위 칸).

### ✅ 베이스 $07 = **0x2100** (Mesen 타일맵 write 트레이스 + 정적 확인)
- 타일맵 기록값 = `tile_index + 0x2100`. 0x2100 = 타일번호부 0x100(256)·palette0·priority1.
- **VRAM 타일 = 256 + tile_index** (표 $C1:D1C3가 준 tile_index를 그대로 더함).
- 표 정적 에뮬(`render_char`)로 메뉴문자열 디코드 검증: は tile_index=0x1A, じ=0x0C(+탁점0x94), め=0x22, か=0x06, ら=0x27, つ=0x12, う=0x03, す=0x0D, け=0x09, 공백=0x00.

### ✅ 메뉴 폰트 = **LZSS 압축, ROM `$D9:0002`** (⚠️이전 "$C9 비압축"은 오식별·폐기)
- 화면=**bgMode 0, 2bpp, chr base=0**(VRAM 타일 0~). 메뉴텍스트 BG tilemap=$4000.
- 폰트 소스 = **`$D9:0002` LZSS 압축**(디컴프레서 `$C0:0D52`→`$C0:0D91`, 출력 항상 `$7F:1000`, 소스 롱포인터 $11-$13, 길이 $05). 압축스트림 3766B(헤더2B 포함) → **디컴프 5984B = 374 타일(2bpp 16B/타일)**.
- **매핑: `$D9` 디컴프 타일 N = tile_index N = VRAM 타일 256+N** (오프셋0 정확확인: $D9 tile0=공백, tile1=あ, tile26(0x1A)=は, tile34(0x22)=め).
- **일반 게임 폰트(전역 공유)** — 히라가나·카타카나·영숫자·마신UI(SP/CN/PW/EXP/LV)·FILE/NODATA/STAGE/MENU 전부 이 시트. 렌더 `tmp/trace/menu2/d9font.png`.
- 편집: `lzss.py` decompress→글리프편집→compress. **무편집 재압축=원본과 정확히 3766B**(코덱 바이트호환, in-place 가능). 단 타일 추가 시 커짐 → in-place 불가 → **재배치+소스포인터/길이 패치 필요**.
- **빈 타일 12개뿐**: tile 0(공백)·252-255·352·368-373. 한글 12자+ 필요 → **부족 → 폰트 확장·재배치 필수**.

### 두번째 렌더러 `$C0:1B4B`(직접타일)·caller `$C0:1BB7`
- $C0:1BB7이 메뉴 렌더 caller(스택파라미터: DBR=$0D,S / Y(문자열)=$0C,S / A(base)=$05,S → JSL $C1965E). base는 상위 전달이라 정적 불가 → 트레이스로 0x2100 확정.

## 한글 주입 계획 (전부 확정, 구현 단계)
편집 대상 3곳 전부 확정:
1. **폰트 $D9:0002**(LZSS): 한글 글리프 타일 추가 → 재압축 → 재배치(소스포인터 패치).
2. **표 $C1:D1C3**(ROM 0x01D1C3): 사용할 SJIS 코드 → 한글 tile_index 매핑 기입.
3. **문자열 $C0:71B9**(ROM 0x0071B9): 한글 매핑 SJIS 코드로 재작성.
- ⚠️ 폰트가 전역 공유라 기존 히라가나 슬롯 덮어쓰기 금지(다른 화면 깨짐) → **미사용 SJIS 코드 + 새 타일** 방식.
- ⚠️ **8×8 셀** 제약 → 한글 가독성: 8×8(1셀), 8×16(body+탁점 2타일), 또는 16×16(2셀×2타일=4타일, 문자당 SJIS 2코드) 중 택. 사용자 결정 필요.

## ✅ 한글 주입 구현 완료·실기 검증 (2026-07-14) — 비파괴 ASM 훅 방식
빌드 `scripts/build_menu.py` → `out/menu_test.smc`. **실기 확인: 처음부터/이어하기/복사/삭제 정상 표시**.

### 왜 단순 방식이 불가했나 (제약 규명)
- **표 body는 8비트** → VRAM 타일 = 0x100+body(0-255) = $D9 타일 0-255만 도달. 그중 안전 빈타일 3개(252-254)뿐.
- **메뉴 폰트 DMA는 256타일만** 로드(src $7F:1000→VRAM tile256, size 4096). VRAM 256-767이 전부 폰트(2회 로드: tile256+tile512). 자유 VRAM = **타일 768-1023**(타일맵은 1024+).

### 구현 (4요소, 원본 폰트·다른 화면 무손상)
1. **한글 글리프 데이터**: Galmuri7 8×8 1bpp(사용자 제공 `8pt_font/*.bin`, 완성형2350 `가`=0)→2bpp(plane0=잉크) 12타일(192B) → 자유 ROM `$C1:9880`.
2. **로더 DMA 주입**: 메뉴 폰트 로더 `$C0:6F43`(디컴프 $D9:0000→$7F:1000, DMA→VRAM tile256). 그 안 `$C0:6F93`(`A9 7F 00 85 07`)를 `JSL $C19940`+NOP로 대체 → 인젝트 루틴 `$C1:9940`이 한글 12타일을 **VRAM 타일 800**으로 별도 DMA(원래 명령 복원 후 RTL).
3. **렌더러 훅**: `$C1:965E`의 `$9696`(`AND #$00FF`)를 `JMP $9843`로 대체 → 훅 `$C1:9843`: **표값 상위바이트(=$06)==0xFE(마커)면 확장경로**(tile=low+0x220+base=VRAM 800+low, 탁점 없음), 아니면 원본 body+탁점 로직. 마커 0xFE는 표 미사용값이라 안전.
4. **표 `$C1:D1C3`**: SJIS `0x8240-0x824B`(SJIS 미정의 갭코드, 안전) 12슬롯(rowbase 0x188, 파일 0x01D34B~) = `0xFE00-0xFE0B`.
5. **문자열 `$C0:71B9`**: 41B 슬롯 in-place.

### ✅ 활성/비활성 색상 = 원본 옵션 컬럼 정렬 (핵심)
- 게임이 **타일맵버퍼 $7E 고정 컬럼에 palette 0/1**을 씀(원본 버퍼 덤프 확인): はじめから(버퍼 col8-13)=**pal0 흰색(활성)**, つづきから/うつす/けす(col14-23,26-28)=**pal1 회색(비활성=세이브없음)**. base는 렌더 시 옵션별로 다르게 전달(pal0=0x2100/pal1=0x2500).
- **한글을 원본 옵션 컬럼에 정렬**해야 색이 옵션단위로 맞음: 처음부터→col8-11(はじめから슬롯), 이어하기→col14-17(つづきから), 복사→col20-21(うつす), 삭제→col26-27(けす). 정렬 안 하면 짧은 한글이 어긋나 엉뚱한 타일이 회색(예: 우기만 흰색 버그).
- **지우기(3타일)→삭제(2타일)**: けす 슬롯 2타일 + 문자열 41B 꽉참 → 완전형 유지하며 2타일 표기(사용자 승인). 정렬 후 삭제도 정상 회색.

### 🔲 남은 과제 (사용자 지정)
- **세이브 데이터 존재 시** 이어하기가 활성(흰색)으로 바뀌는지, 그때 글자 표기·색상 정상인지 실기 검증(현재는 세이브 없어 전부 비활성). 세이브 후 게임 진행해 확인 필요.
- (선택) 지우기(3타일) 유지 원하면 문자열 재배치+호출부 포인터 패치.
- FILE1~4·NO DATA·MENU 등 나머지 SJIS UI 한글화(동일 기법 확장 가능).
- 캐논 ROM(out/wgp2_kr.smc, 대사패치)에 메뉴 패치 통합.
