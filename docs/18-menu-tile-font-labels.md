# 17 · 메뉴 소형 타일폰트 라벨 (System ④) — 월드맵 X메뉴·조작방법 한글화

> 규명 완료 2026-07-21. docs/11(메뉴폰트·렌더러)·docs/12(SJIS 한글 인프라)의 후속. 구현 대상 = 월드맵 X버튼 메뉴 + 조작방법(そうさマニュアル) 튜토리얼 라벨.

## 규명 결과 — 네 번째 텍스트 경로

게임의 텍스트 렌더는 4경로다. 이 문서 대상은 ④.

| 시스템 | 렌더러 | 폰트 | 예시 |
|---|---|---|---|
| ① 압축 글리프 | 파서 $C1:9554 | $CA 16×16(한글 교체됨) | 대사·스토리·조작방법 **나레이션**(이미 번역) |
| ② 비압축 SJIS | $C1:965E(SJIS→표 $C1:D1C3) | $D9 8×8(한글=VRAM800) | 파츠명·인명·저장메뉴(docs/12 한글화) |
| ③ LZSS 그래픽 | BG DMA | 타일=그림 | 챕터타이틀·큰 로고 |
| **④ 직접 타일** | **$C0:1B4B(직접 타일인덱스)** | **$D9 8×8**(②와 동일 폰트) | **월드맵 X메뉴·조작방법 라벨·세팅창·개러지 메뉴** |

### 핵심: ④는 "직접 타일인덱스 문자열"

- **폰트**: docs/11 확정 — `$D9:0002` LZSS → 디컴프 $7F:1000 → VRAM 타일256, **8×8 2bpp, 374타일**(전역 공유: 카나·라틴·숫자·SP/CN/PW/DF/WT/DP/BP/LV/EXP·마신UI). 빈타일 ~12개뿐.
- **렌더러 `$C0:1B4B`**(직접 렌더러, IDA 디스어셈블 완료): 문자열을 1바이트씩 읽어 `tile = byte + base` 를 타일맵버퍼 `$7E:0000,X`에 기록. `byte==0x00`→메인행 종료 후 오버레이행(`$7D:FFC0,X`, 탁점/제어 jmp표 $C0:1BB0) 처리. `byte==0xFF`→빈타일(0). base(=DP `$00`의 상위워드)는 호출자가 전달(예 0x2100 = VRAM타일256|pal|prio).
  - ②의 $C1:965E와 **동형 구조**(같은 base·같은 $7E/$7D 버퍼). 차이 = ②는 SJIS 2바이트→표변환, ④는 타일인덱스 직접.
- **라벨 = $D9 폰트의 타일번호 시퀀스**. SJIS도 System①도 아니라서 그 인코딩 검색엔 안 잡혔음. **변환표 $C1:D1C3 역산**(SJIS→tile_index를 뒤집어 tile_index→문자)으로 전량 디코드 성공.

### 라벨 문자열표 위치 (직접타일, null 종료 + 위치/제어바이트)

| 화면 | 주소 | 내용 |
|---|---|---|
| 조작방법 튜토리얼 8p | **$C7:B180~B460** | 목차(P2~P7)·버튼캡션(マシンきりかえ/ウィンドウ/けってい/キャンセル…)·파라미터(SP スピードせいのう…)·모드타이틀 |
| 세팅 창메뉴(박스) | **$C1:C6D0~C7A0** | イージーセッティング/セッティングセーブ/ロード 등(박스문자 $E0~$E7 포함) |
| 개러지 메뉴 | **$CE:46xx~** | そうさ 등 |

- 디코드 방법(오프라인): 변환표 $C1:D1C3(파일 0x01D1C3) 2단계 역산 → tile_index→문자 맵 → 바이트열 디코드.

## 한글 렌더 방식 (사용자 확정: docs/12 8pt 한글 재사용 + 마커훅)

②(docs/12)가 이미 **8pt 한글 200음절을 VRAM 800~999**에 올리고, 렌더러 $C1:965E에 **0xFE 마커 훅**($C1:9843)을 심어 "표값 상위==0xFE면 tile=low+0x220+base=VRAM800+low"로 처리한다. ④에도 **동일 마커 훅**을 이식한다.

- **$C0:1B4B 마커 훅**: 메인행 루프에서 `byte==0xFE`면 다음 바이트=한글 음절 인덱스 → `tile = idx + 0x220 + base`(VRAM800+idx), 오버레이 없음. `byte==0xFF`(빈타일)·`0x00`(종료) 기존 로직 유지. KOR_ADD=0x220은 ②와 동일.
- **음절 풀 공유**: ②의 `syls`(build_sjis.py) 순서 그대로 사용 → ④ 마커 idx = 같은 음절 인덱스. 라벨 한글은 이 풀에 append.
- **VRAM 예산**: 상한 **224**(VRAM 800~1023). 현재 ② 사용 **200** + 라벨 순증 **24** = **정확히 224**(여유 0). ⚠️ 라벨 번역은 타이트하게 유지(순증 음절 늘면 초과 → 224 재RE 필요). [[sjis-0x86-expansion-suspect]]의 0x86 확장이 189→224 슬롯을 이미 열어둠.
- **폰트로더 훅(미검증)**: docs/12는 저장메뉴 $C0:6F93 + 개러지/선택 $C1:1EFB·$C1:307F 3곳에서 한글을 VRAM800에 DMA. **조작방법 튜토리얼 화면이 이 로더 중 하나를 쓰는지 미확인** → 실기 테스트에서 라벨이 빈칸/깨지면 그 화면 폰트로더를 추가 훅(트레이스 `scripts/lua/trace_menu_font.lua` 또는 IDA). **실기 검증 게이트에서 확정.**

## 라벨 번역 (glossary 준수)

- 규칙: 2글자 스탯 라벨(SP/CN/PW/DF/WT/DP/BP)은 원문 유지, 설명만 번역. 세팅 서브메뉴(イージー/マニュアル)는 **원문 유지 = 이지/매뉴얼 세팅**(사용자 지시, System① $C1:C868 기존 번역 그대로).
- 초안(순증 24음절): 목차·맵 모드·세팅 모드·파라미터·그리드변경 모드·레이스 모드·WGP출전 모드 / 액션·이동·매뉴얼·윈도우·대시 / 머신전환·파츠선택·결정·취소·머신선택·종료 / 스피드성능·코너링성능·파워성능·다운포스성능·무게·내구력·폭주포인트 / 체인지·스티어링·포즈·톱체인지·회복·폭주·레이스아이템·선택 / 아이템·조작방법·용어집·지도.
- ⚠️ 슬롯 제약: ④ 라벨은 in-place(null 종료 고정슬롯). 마커 인코딩은 한글 1음절=2바이트(0xFE+idx), 원본은 타일 1개=1바이트 → 긴 한글 라벨은 슬롯 초과 가능 → 축약 or 재배치.

## $C0:1B4B 마커 훅 (어셈블 완료 — 원시 바이트 검증)

렌더러 원시 바이트($C0:1B4B~):
```
1B5C: B7 0F        LDA [$0F],Y     ; 메인루프: 바이트 읽기(포인터 dp $0F, 24bit)
1B5E: C8           INY
1B5F: 29 FF 00     AND #$00FF
1B62: F0 13        BEQ $1B77       ; 0 → 오버레이행
1B64: C9 FF 00     CMP #$00FF      ; ← 이 8바이트를 훅 JML로 치환
1B67: D0 03        BNE $1B6C
1B69: A9 00 00     LDA #$0000
1B6C: 18 65 01     CLC; ADC $01    ; +base(dp $01), 훅이 재사용
1B6F: 9F 00 00 7E  STA $7E0000,X   ; 타일맵버퍼
1B73: E8 E8 80 E5  INX;INX;BRA $1B5C
1B77: (오버레이행 B7 0F… CMP #3 jmp표 $1BB0 / +base → $7DFFC0,X)
```

**패치** @ $C0:1B64 (8바이트 `C9 FF 00 D0 03 A9 00 00` → JML훅+NOP): `5C <lo> <hi> C1  EA EA EA EA`.

**훅 루틴 (31B, 자유 ROM $C1:xxxx)**:
```
C9 FE 00      CMP #$00FE          ; 마커?
F0 0C         BEQ marker
C9 FF 00      CMP #$00FF          ; 빈타일?
D0 03         BNE notff
A9 00 00      LDA #$0000
notff: 5C 6C 1B C0   JML $C01B6C  ; 정상: CLC;ADC base;STA (A=byte or 0)
marker: B7 0F        LDA [$0F],Y  ; 다음바이트=한글 음절 idx
C8            INY
29 FF 00      AND #$00FF
18            CLC
69 20 02      ADC #$0220          ; +0x220 (VRAM800-256), KOR_ADD
5C 6C 1B C0   JML $C01B6C         ; CLC;ADC base → idx+0x220+base = VRAM800+idx
```
- 2바이트 소비(0xFE+idx), 1타일 기록. 오버레이 없음. base=0x2100이면 VRAM 800+idx|attrs.
- 자유 ROM 위치는 build 통합 시 확정(build_sjis $C1 free 영역, 0xFF fill assert). build_sjis 훅들($C1:9843/9940/9980/9A00)과 비겹침.

## 2026-07-22 실제 구현 정정 — 문맥별 원본 폰트 재압축

초기의 마커 훅 안은 사용하지 않았다. 최종 `build_menu4_reclean.py`는 원본 `$D9:0000`을 화면 문맥별로 복제해 필요한 타일만 8pt 한글로 바꾸고, 기존 로더 포인터만 리다이렉트한다. 코드 훅·NMI·추가 DMA는 없다.

| 문맥 | 자원 | 로더 | 비고 |
|---|---|---|---|
| 월드맵 X메뉴·튜토리얼·다음 LV·퀴즈 상태줄 | `$C7:E000` | `$C0:94D4` 및 튜토리얼 포인터 | 82음절, `$F4-$F8` 고정 그래픽 포함 |
| 저장 삭제·덮어쓰기 확인창 | `$C7:C000` | `$C0:6F4B` | `괜찮습니까？ / 예 / 아니오` 전용 8타일 |
| 용어집 | `$C2:D448` | `$C3:9DC5` | 문맥 전용 타일 매핑 |
| 지도 | `$C2:E280` | `$C3:A835` | 문맥 전용 타일 매핑 |
| 이지·수동 세팅 X메뉴·다음 LV | `$C7:D000` | `$C1:1EBB` / `$C1:303F` | 교체된 원문 4행의 가나 14타일 + `$F4-$F8` 전용 그래픽만 교체 |

저장 확인창 `$C0:7841`은 월드맵과 같은 직접 렌더러를 쓰지만, 저장 관리 화면 진입 시 `$C0:6F4B`가 원본 `$D9`를 별도로 다시 올린다. 따라서 문자열 바이트만 바꾸면 `よろすちな？ / ぬ / ュちね`처럼 원본 가나가 표시된다. 저장 화면의 다른 직접 프로그램 네 구간에서 사용하지 않는 타일 `$3F-$46`만 전용 자원에서 교체하고, 빌드가 충돌 여부를 검사한다.

수동 세팅은 한 프레임의 타일맵에 보이지 않는 슬롯을 '미사용'으로 간주하면 안 된다. 초기 구현이 `$80/$82/$85/$89` 등을 재사용해 동적 모델명의 `S/V/Z`가 한글로 변하는 회귀가 발생했다. 수정본은 한글로 완전 교체되는 원문 라벨 4행이 쓰던 가나 슬롯에서만 14개를 회수하고, `$70-$9F` 영문·숫자 대역이 원본과 바이트 단위로 일치하는지 최종 ROM 검증기가 확인한다.

`$F4-$F8` 타일 위치는 이지·수동 세팅에서 같지만 폰트 로더가 둘이다. 수동 화면은 `$C1:1EBB`에서 0x1400B, 이지 화면은 `$C1:303F`에서 0x1600B를 VRAM 타일 256에 올린다. 초기 수정은 전자만 `$C7:D000`으로 리다이렉트해 이지 화면에서 `次のLVまで`가 남았다. 현재는 두 소스 포인터를 모두 같은 파생 자원으로 연결하며, 각 DMA 길이가 `$F4-$F8`을 포함하는지 빌드와 최종 ROM 검증기가 검사한다.
