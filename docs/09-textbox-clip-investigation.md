- id=444/463/486 `（START로 종료）` → 화면 "RT로 종료" (앞 `（STA` 잘림)

## 정적 역공학으로 확정한 렌더 경로 (bank $C0)
- 글리프 렌더러 **`$C0:6827`** (스택프레임 진입, 인자 = 스택상 `$22`(라인버퍼 base)·`$24`(펜 X px)·`$26`(글리프 인덱스)).
  - 폰트 페치: `LDX $03; LDA $CA1137,X` ($03 = base03($26)). base03 = `16*((n&7)+2*(n&~7))` 확인.
  - 목적지: `Y = $22 + (($24 & 0xF8)>>2)`, 서브타일 x=`$24&7` → `JSR ($6A54,X)` 시프트.
  - 펜 전진: `LDX $26; LDA $CA9137,X`(폭테이블) → `$24 += width` (유일 STA $24 @ `$C0:68A4`).
- 렌더러 호출처 3곳: **`$C1:9592`(파서=레이스/스토리 673 경로)**, `$C0:66CD`, `$C0:4091`.
- `$22`/`$24`(라인 base·펜 X 원점)는 렌더러 내부에서 초기화 안 함 → **호출자(파서/펌프)가 라인 시작마다 세팅**. 이 원점 계산이 고정값인지 컨텐츠 의존(센터링)인지가 잘림 원인의 핵심.
- 기존 render_trace(PoC 초상화 상자, "부천 레트로 흥해라")에서 첫 글리프 dst offset ≈0x10 → 원점 X ≈ 64px(0x40) = 초상화 폭만큼 우측 시작. 즉 상자 유형별 원점 X가 다름.
