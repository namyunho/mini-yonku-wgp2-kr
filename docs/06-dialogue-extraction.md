# 06 · 대사 추출 — 초기 탐색 기록

> **문서 성격: 역사적 스냅샷.** 이 문서는 포인터 구조를 알기 전 전역 스캔으로 최초
> 651개를 찾아낸 과정을 보존한다. 현재 완전성·범위·수치는
> [`07-dialogue-completeness.md`](07-dialogue-completeness.md)와
> `assets/translations/dialogue.json`이 정본이다.

인코딩 확정(docs/04) 뒤 종료자 워크와 일본어 응집도 판정으로 정적 대사 후보를 전역 탐색했다.
처음에는 `$C7`의 레이스 대사 232개만 확인했으나, ROM 전체를 다시 스캔해 `$D0`의
스토리/배틀 대사 404개와 `$C1`의 포메이션 대사 15개를 추가로 발견했다.

## 당시 추출 결과

| 블록 | 위치(파일) | 당시 메시지 | 내용 |
|---|---|---:|---|
| `c7_race` | `$C7:89E2` 이후 | 232 | 레이스 중계·조작 안내 |
| `d0_story` | `$D0:C80B` 이후 | 404 | 스토리·배틀 대사 |
| `c1_form` | `$C1:CFAF` 이후 | 15 | 포메이션 메뉴 |
| 합계 |  | **651** | 초기 탐색 스냅샷 |

당시 651개 전부에서 `encode(parse(text)) == raw_hex == ROM`을 통과했고, 미매핑 글리프와
미정의 opcode는 없었다. 이 결과는 인코딩 모델과 종료자 워크가 맞다는 증거였지만,
“작은 파편 블록이 더 없는가”를 증명하지는 못했다.

## 탐색 방법

1. 미정의 opcode `0x08–0x0F` 없이 정상 종료되는 긴 연속 구간을 후보로 찾았다.
2. 평균 글자 수, 히라가나 비율, 조사 밀도로 코드·그래픽·카나 테이블을 걸러냈다.
3. 후보를 실제 일본어로 디코드해 사람이 판독했다.
4. `$C7`은 뱅크 `$C3`의 VM opcode `EF CB`/`D4 CB` 뒤 포인터가 메시지 경계에
   착지하는지 교차 확인했다.

휴리스틱은 대형 연속 블록 발견에는 유효했지만, `$C1`처럼 텍스트와 바이너리가 섞인 작은
클러스터의 완전성에는 적합하지 않았다. 이후 docs/07에서 파서 `$C1:9554`의 호출처 7곳을
전수 역추적해 정적 텍스트 뱅크가 `$C7/$D0/$C1`뿐임을 닫고, 누락된 `$C1` 문자열을 복구했다.

## 현재 결과로의 연결

| 항목 | 초기 스냅샷 | 현재 정본 |
|---|---:|---:|
| 정적 대사 | 651 | **681** |
| `c7_race` | 232 | 232 |
| `d0_story` | 404 | 404 |
| `c1_ui` | 15 | **45** |
| 완전성 근거 | 휴리스틱 스캔 | 단일 퍼널 파서 호출처·포인터 전수 열거 |

추출기와 회귀 게이트는 `scripts/extract_dialogue.py`, `scripts/final_catalog.py`,
`scripts/test_roundtrip.py`이며, 현재 기계 판독 정본은
`assets/translations/dialogue.json`과 `assets/translations/pointer_catalog.json`이다.
