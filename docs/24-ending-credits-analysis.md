# 24 · 실제 엔딩 크레딧·베스트타임 분석

> 2026-07-23 정적 역공학 결과. 이 문서의 “엔딩”은
> `out/wgp2_kr_017.png`~`023.png`에 찍힌 최종 스크롤 크레딧과
> 베스트타임 화면을 뜻한다. `build_ending_logo.py`가 다루는
> `VICTORYS` 에피소드 인터미션과 부팅 직후 저작권 크레딧은 별개다.

## 결론

- 실제 엔딩 문자는 그림 한 장이나 SJIS 문자열이 아니라
  **8×8 2bpp 소형 글꼴의 직접 타일맵 명령열**이다.
- 크레딧의 39개 고정 문구와 이름, 베스트타임 제목·코스명·`びょう`는
  현재 행 폭 안에서 한글로 바꿨다. 완역 45행 중 폭을 넘거나 축약한 행은 0개다.
- 크레딧·베스트타임과 현지화 메시지에 필요한 고유 한글 음절은 136개이고,
  프로젝트의 8pt 글꼴이 136개를 모두 보유한다. 쉼표·마침표·가운데점
  3개를 더해도 엔딩 전용 256타일 자원에 들어간다.
- 베스트타임 네 줄의 숫자는 저장 데이터에서 런타임에 덮어쓴다.
  숫자 코드는 보존하고 제목·코스명·단위만 번역하면 기록 계산에는 영향이 없다.
- 사용자 승인 현지화 메시지 12행을 총괄 프로듀서 이름과 베스트타임 사이의
  원래 빈 논리행에 넣었다. 명령열은 같은 C7 뱅크의 검증된 자유 영역으로
  재배치했으며, 154레코드 순서와 스크롤 타이밍은 유지된다. 구현 뒤에도
  123바이트, 최대 폭 31타일 기준 1개 논리행의 추가 여유가 남는다.

## 실제 데이터와 소비 코드

| 항목 | 주소 | 확인 내용 |
|---|---:|---|
| 원본·그림자 명령열 | `$C7:A2A5~B182` | 3,806바이트. 원래 45행 번역을 같은 길이로 보존 |
| 활성 엔딩 명령열 | `$C7:EE7B~FF84` | 현지화 메시지를 포함한 4,362바이트, 종료어 `$FFFF` 포함 |
| 초기 포인터 | `$C3:8275` | `LDA #$EE7B` → 활성 명령열 시작 주소 저장 |
| 행 인터프리터 | `$C3:8285` | 행 길이만큼 포인터 전진, 첫 타일값 0~5로 처리기 분기 |
| 일반 행 처리 | `$C3:83BC` | C7의 타일맵 단어를 현재 스크롤 행의 VRAM으로 전송 |
| 베스트타임 처리 | `$C3:8409/8417/8425/8433` | 코스별 저장 기록 선택 후 숫자 덮어쓰기 |
| 숫자 변환 | `$C3:8441` | 초·백분의 초를 10으로 나누어 네 숫자 타일 생성 |
| 엔딩 글꼴 실진입 로더 | 호출 `$C3:6B31`, 포인터 `$C3:6B35` | 실제 엔딩 첫 진입에서 원본 `$D9:0000`을 해제해 `$7F:1000`에 둔 뒤 첫 0x1000바이트(256타일) DMA |
| 엔딩 글꼴 대체 로더 | 호출 `$C3:6CFF`, 포인터 `$C3:6D03` | 같은 크레딧 오브젝트의 대체 초기화·재진입 경로. 두 포인터를 모두 전용 글꼴로 통일 |
| 엔딩 오브젝트 기술자 | `$C3:E343` | 초기화 주소 `$8275`를 참조하여 위 두 경로가 같은 화면임을 확인 |

명령열은 154개 행 레코드와 종료어로 구성된다.

- 실제 문자가 있는 일반 상단/본문 쌍: 41쌍
- 베스트타임 상단/본문 쌍: 4쌍
- 완전 공백인 길이 0 상단/본문 쌍: 31쌍
- 종료 직전 폭 5의 공백 상단/본문 쌍: 1쌍
- 합계: 77쌍

일본어 탁점·반탁점은 본문 타일 위의 별도 상단 행에 `$94/$95`로 놓인다.
한글은 이 오버레이가 필요 없으므로 본문 행만으로 표시할 수 있다.

레코드의 첫 타일값은 화면 문자이면서 처리기 번호다.
일반 행은 0, 베스트타임 네 줄은 각각 1~4다. 원문에서는 이 네 타일이
`あ/い/う/え`로도 보인다. 구현본은 코드 1~4를 보존하고 엔딩 전용 글꼴의
해당 네 타일만 `가/나/다/라`로 바꿨다. 따라서 분기는 원형 그대로다.

## 베스트타임 행

베스트타임 제목은 6타일 슬롯이다.

- `ベストタイム` 6타일 → `베스트타임` 5타일

각 코스 행의 고정 배치는 다음과 같다.

| 열 | 용도 |
|---:|---|
| 0 | 처리기 선택용 1~4. 보존 필요 |
| 1~5 | 공백 |
| 6~17 | 코스명 12타일 |
| 18~19 | 런타임 초 두 자리 |
| 20~22 | 고정 단위 `びょう` |
| 23~24 | 런타임 백분의 초 두 자리 |

| 원문 | 코스명 슬롯 | 후보 번역 | 필요 타일 |
|---|---:|---|---:|
| `たかばやま` | 12 | `타카바산` | 4 |
| `ふうりんサーキット` | 12 | `풍륜 서킷` | 5 |
| `しょうがっこう` | 12 | `초등학교` | 4 |
| `つちやレースじょう` | 12 | `츠치야 경기장` | 7 |
| `びょう` | 3 | `초` | 1 |

숫자는 `$C3:8441`이 타일 `$A0~A9`로 매번 덮어쓴다. 따라서 `$A0~A9`를
한글 글리프로 재사용하지 않고 코스명·단위만 패치해야 한다.

## 글꼴 타일 예산

원본 `$D9:0000`의 해제 크기는 5,984바이트(2bpp 374타일)지만,
엔딩 로더가 올리는 범위는 첫 4,096바이트, 즉 타일 0~255다.
공용 원본을 직접 고치지 않고 엔딩 전용 사본을 만들고
실제 첫 진입 `$C3:6B35`와 대체 진입 `$C3:6D03`의 소스 포인터를
함께 바꾸는 방식이 안전하다.

보수적으로 다음 슬롯을 모두 보존해도 된다.

- 명령·공백 타일 `$00~$05`: 6개
- 런타임 숫자 `$A0~$A9`: 10개
- 라틴 대문자 `$70~$89`: 26개

256개 중 42개를 보존하면 214개가 남는다.
크레딧·베스트타임에 필요한 고유 한글은 86음절이고, 현지화 메시지를
합치면 136음절이다. `8pt_font/font-007242d37349daf3_glyph_map.json`에서
136개 모두 확인됐다.
구현은 처리기 `$00~05`와 라틴·원본 숫자·탁점·런타임 숫자가 모인
`$70~A9`를 보수적으로 모두 보존해 192개 후보 타일만 사용한다.
쉼표·마침표·가운데점 3개까지 합친 139타일을 사용하므로
구현 뒤에도 후보 타일 53개가 남는다.

엔딩 전용 글꼴은 `$D9:E000`에 두었고, 해제 크기 5,984바이트다.
`$C3:6B35/$C3:6D03`의 인라인 포인터를 `$D9:0000`에서
`$D9:E000`으로 바꿔 공용 소형 글꼴은 수정하지 않았다.

2026-07-23 실기 트레이스
`scripts/lua/trace_ending_credits_font.lua`에서 최초 구현의
WRAM `$7F:1000`과 VRAM word `$4000`이 원본 글꼴 첫 4096바이트와
완전히 일치함을 확인했다. 번역 명령열은 `$C7:A2A5`에서 정상 소비됐지만,
실제 첫 진입이 `$C3:6B31/$C3:6B35`를 사용한 것이 원인이었다.
같은 오브젝트 기술자 `$C3:E343`을 참조한다는 정적 근거만으로
대체 초기화 `$C3:6CFF/$C3:6D03`을 유일한 로더로 판단한 기존 결론을
이 동적 증거로 정정했다.

## 현지화 메시지 삽입

현재 최종 ROM에서 `$C7:EE7B~FFFF`는 4,485바이트의 연속 자유 영역이다.
원본 명령열 3,806바이트를 이곳으로 옮기면 679바이트를 추가할 수 있다.

- 시작 포인터 `$C3:8275`의 `$A2A5`를 `$EE7B`로 바꾼다.
- 인터프리터는 별도 길이 상수 없이 `$FFFF`까지 읽는다.
- 종료 플래그도 `$FFFF`를 만났을 때만 세운다.
- 한글 16픽셀 논리행을 “빈 상단 레코드 + 최대 31타일 본문 레코드”로
  만들면 최대 66바이트다.

명령열의 길이 0 공백 논리행 31개 가운데 총괄 프로듀서 이름 뒤의
pair 49, 51~61에 다음 12행을 넣었다.
33타일인 작품명 나열만 쉼표 뒤에서
두 줄로 나눴고 글자는 줄이거나 바꾸지 않았다.

```text
한글번역 피아

SPECIAL THANKS TO
날아라 부메랑, 우리는 챔피언,
폭주형제 WGP·MAX...
미니카 하나로 밤새 가슴 설레어하던
그 시절의 우리들.
세월이 흘러 어른이 되어버린
세상의 모든 챔피언들에게 이 번역을 바칩니다.
부천 레트로게임 수다방,
한마루에서 응원해주신 분들
이 게임을 처음 하게 해줬던 성현이
모두 감사드립니다.
```

원본 154레코드를 추가하거나 삭제하지 않고 빈 레코드 12개에 본문만
채웠으므로 스크롤 타이밍은 그대로다. `한글번역 피아`와
`SPECIAL THANKS TO` 사이의 pair 50, 베스트타임 앞의 pair 62를
공백으로 남겼으며 모든 메시지 행을 가운데 정렬했다.

- 활성 명령열: **4,362바이트**
- 재배치 영역의 남은 공간: **123바이트**
- 추가 가능한 최대 폭 독립행: **1행**
- 남은 길이 0 공백행: **19행**
- 남은 글꼴 후보: **53타일**

## 구현 시 회귀 게이트

1. 원본 엔딩 명령열 154레코드를 전량 파싱하고 종료어 위치를 검증한다.
2. 일반 행의 첫 타일 0과 베스트타임 행의 1~4를 보존한다.
3. 런타임 숫자 타일 `$A0~A9`와 숫자 기록 주소를 보존한다.
4. 전용 글꼴은 원본 `$D9:0000`을 수정하지 않고 별도 자원으로 만든다.
5. 스트림을 재배치하면 C7 뱅크 안에 두고 `$FFFF` 종료까지 다시 파싱한다.
6. `scripts/test_roundtrip.py`, 전체 빌드, 2MiB 크기, BPS 적용 동일성 검사를 통과한다.
7. Mesen2에서 크레딧 전체 스크롤, 네 코스의 서로 다른 저장 기록,
   추가 메시지의 진입·종료 타이밍을 실기로 확인한다.
