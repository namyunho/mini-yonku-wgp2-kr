# 08 · 어드벤처/스토리 텍스트 엔진 — 🟡 발견·해독 진행 중

## 요약 (왜 중요한가)

docs/07의 "정적 대사 673 완전성"은 **파서 `$C1:9554` 한 시스템에 국한**된 것이었고, **화면 전체 텍스트의 완전성이 아니었다.** 게임의 **어드벤처/스토리 모드 대사는 완전히 별개의 압축 텍스트 엔진**이 담당한다(673 밖).

- 673(레이스·배틀·메뉴) 재삽입은 완료·검증됨(→ docs/06,07, `scripts/build_patch.py`).
- **어드벤처 텍스트는 미해독·미번역.** 폰트 시트 $CA를 공유하므로, 673만 재삽입한 현재 빌드에서 **어드벤처가 화면에 깨져 보인다**(카나·저인덱스 한자 슬롯을 한글로 덮은 탓). → 어드벤처까지 처리해야 게임이 온전.

## 동적 트레이스로 확정한 사실 (Mesen Lua)

파서 훅(`trace_text.lua`)으로 어드벤처를 플레이 → 파서 `$C1:9554`는 **어드벤처 대사를 미경유**(호출처 $C3789C·$D0516D 뿐). 즉 별개 엔진.

| 항목 | 값 | 근거 |
|---|---|---|
| **엔진(디코더 루프)** | **`$C0:39E0`~ (읽기 `$C0:39E7`, `$C0:3A05`, 딕셔너리 페치 `$C0:3A4A`, 출력 `$C0:3A4D`)** | `trace_c6.lua`, `trace_advptr.lua` |
| **소스 텍스트 뱅크** | **$C5**(주력, 최소 `$5950..$60C2`) · **$C4**(`$3429..$34D6+`) — 더 있을 것 | `trace_advptr.lua`(adv_ptr.txt) |
| **소스 포인터** | DP **`$11-$13`** 24비트 롱포인터 (`LDA [$11]`로 1바이트씩, `INC $11`/`$13`) | 디스어셈블 |
| **딕셔너리** | **`$C6:7C73`** (파서 글리프표로 보면 `　0123456789「」『』あいうえお…JKLM…！？。・…一九七十…` = charmap/프래그먼트 풀) | `trace_c6.lua`(c6_read.txt), 엔진 `LDA $7C73,X` |
| **출력 버퍼** | `STA [$15],Y` (DP `$15-$17` 롱포인터) → 디코드된 글리프 스트림 | 디스어셈블 |

### 압축(디코드) 개략
엔진 내부 루프(부분 디스어셈블, M/X 폭 토글로 정적 해석은 주의):
```
LDA [$11]        ; 소스값 A (16비트 추정) 읽고 $11 전진
... (제어 처리: $3A0D/$3A17 분기)
AND #$0007 ; INC ; STA $01   ; len = (A & 7) + 1
LSR LSR LSR ; TAX            ; X = A >> 3  (딕셔너리 인덱스)
loop $01회: LDA $7C73,X ; STA [$15],Y ; INX ; INY   ; 프래그먼트 확장
```
→ **소스 = (딕셔너리 인덱스, 길이) 참조 스트림**, 딕셔너리 `$C6:7C73`에서 (len)바이트 프래그먼트를 확장해 출력 버퍼에 씀. 사전 압축(dictionary/LZ 유사).
파서 글리프표로 소스를 직접 디코드하면 깨짐(단어 조각만 식별: ワールドグランプリ·チャンピオン·GPチップ·ディアナ·ビクトリーズ·われわれ).

## 다음 세션 할 일 (해독 완결 경로)

1. **출력/소스 동시 캡처**로 코덱 검증: `scripts/lua/trace_advout.lua`(엔진 `$C03A4D` 출력·`$C039E7` 소스 훅) 실행 → 어드벤처 대화 1~2장면 → `tmp/trace/adv_io.txt`에 입력바이트열·출력글리프열 확보. 출력 글리프열을 파서 글리프표로 렌더 → **정확한 일본어 대사 확인**(예: "土屋博士「やあみんな{nl}ひさしぶりだ").
2. **딕셔너리 `$C6:7C73` 전체 덤프** + 엔진 정밀 디스어셈블(--m/--x 폭 추적)로 **오프라인 Python 디코더** 구현, adv_io.txt로 검증.
3. **어드벤처 텍스트 전량 추출**(소스 뱅크 $C4/$C5+ 범위 확정 — 포인터 테이블/스크립트 진입 파악) + 분량 집계.
4. **재인코딩 가능성 검증**: 딕셔너리 재구성 or 압축 우회(비압축 재기록 + 엔진 우회 패치) 설계 → 재삽입 방식 결정.
5. 이후 번역·재삽입.

## 관련 트레이스 스크립트 (`scripts/lua/`)
- `trace_text.lua` — 파서 $C1:9554 훅(대사 주소·호출처). 어드벤처 미경유 확인용.
- `trace_c6.lua` — $C6 읽기 PC 훅(딕셔너리 접근 규명).
- `trace_advptr.lua` — 어드벤처 소스 포인터 $11-$13 캡처(소스 뱅크·범위).
- `trace_advout.lua` — 디코드 출력·소스 바이트 캡처(코덱 역공학용).
- 실행: `Mesen.exe "<원본ROM>" "<script.lua>"` (원본 ROM으로 — 일본어 정상 표시). 산출 `tmp/trace/*.txt`.
