# 08 · 어드벤처/스토리 텍스트 엔진 — ✅ 코덱 완전 해독·오프라인 검증

## ✅ 2026-07-12 코덱 완전 해독 (엔진 $C0:39E0 정밀 디스어셈블 + 오프라인 디코더 검증)

`$C0:39E0` 디코더를 **손으로 정밀 디스어셈블**(선형 디스어셈블러는 M/X 폭 토글·16비트 LDY로 오싱크 → 원바이트 덤프로 재구성)해 코덱을 완전 확정. **오프라인 파이썬 디코더 `scripts/adv_codec.py`가 $C5:5950부터 실제 어드벤처 오프닝을 정확히 복원**(검증됨):
> 第1回WGP終了から1ヵ月後… / 土屋博士「やあみんな{nl}ひさしぶりだね / 豪「ああ！それで博士　話ってなんだ？ / 烈「第2回ワールドグランプリの{nl}ことじゃないですか？ / 藤吉「メンバーは当然{nl}この五人でゲスな？ / …

### 확정 코덱 (플래그바이트 LZ + 딕셔너리)
소스 = `[$11-$13]` 롱포인터에서 바이트 단위로 읽는 **단일 인터리브 스트림**. 플래그 비트 버퍼 `$03`(16비트, 상위바이트 0xFF 센티넬)이 8비트 소진 시 소스에서 플래그 바이트 1개를 리필(**LSB first**):
- **플래그 비트 0 = 리터럴**: 소스 1바이트를 그대로 출력.
- **플래그 비트 1 = 딕셔너리 참조**: 소스 2바이트(**LE**) = word → `index = word>>3`, `len = (word&7)+1` → 딕셔너리 `$C6:7C73[index]`에서 **len 바이트**를 출력에 복사.
- `Y(출력수) == $05(목표 글리프수)` 될 때까지 반복.

**출력 바이트열 = 파서(1바이트 가변길이) 인코딩과 동일**: `0x00`=끝, `0x01-04`=2바이트 글리프 프리픽스(빅엔디언, glyph=(b<<8|next)−0x10), `0x05`=개행, `0x06/07`=제어(1인자), 그 외 glyph=byte−0x10. → **어드벤처도 673과 같은 글리프표·폰트시트 $CA를 공유**. 딕셔너리 저부(0x00-0xFF)는 **아이덴티티 charmap**(dict[i]=i, 단일글리프용), 그 위는 실제 어구 프래그먼트(土屋·校長·ファイター·豪·烈·たまみ·タモツ·森…).

### 손 디스어셈블 (핵심 루프, $C0:39FC=루프탑, M=0/X16)
```
39F7 STZ $03 ; 39F9 LDY #$0000
39FC LSR $03                        ; 루프탑
39FE LDA $03 ; BIT #$0100 ; BNE $3A12   ; 플래그버퍼 남았으면 리필 스킵
3A05 LDA [$11] ; INC ptr ; ORA #$FF00 ; STA $03   ; 리필: 새 플래그바이트|센티넬
3A12 LDA $03 ; BIT #$0001 ; BNE $3A2E   ; 비트0=1 → 딕셔너리 참조
3A19 SEP#20 ; LDA [$11] ; STA [$15],Y ; REP#20 ; INC ptr ; INY   ; (비트0=0) 리터럴 1B
3A28 CPY $05 ; BCC $39FC ; BRA $3A5B(끝)
3A2E LDA [$11] ; INC ptr×2           ; 참조: 소스 2바이트(word)
3A3C PHA ; AND #$0007 ; INC A ; STA $01 ; PLA ; LSR×3 ; TAX   ; len=(w&7)+1, X=w>>3
3A48 SEP#20 ; LDA $7C73,X ; STA [$15],Y ; REP#20 ; INX ; INY ; DEC $01 ; BNE $3A48
3A57 CPY $05 ; BCC $39FC ; (else) 3A5B PLA×3;PLY;PLX;PLD;PLB;RTL
```

## ✅ 2026-07-13 메시지 프레이밍/제어코드 역공학 (진행 — 씬 인터프리터 일부)

디컴프 출력 스트림을 연속 디코드해 **제어 어휘 확정**(출력 40KB 분석):
- **제어코드 = 인코딩 프리픽스 `0x04`(2바이트 `04 xx`) → 글리프인덱스 0x3F0+**. 글리프표는 0x3EF까지(1008개)라 **0x3F0+는 전부 제어**(프리픽스 0x01–0x03 = 실제 한자 글리프). ⇒ 스트림에서 `04 xx`면 제어, `01/02/03 xx`·단일 0x10–0xFF면 텍스트로 **깔끔히 판별 가능**.
- **핵심 제어**: `0x3F6`(×341, **메시지 이벤트 끝**), `0x3F5`(×339, **페이지/계속 브레이크**), `0x3F0`(×259, **박스/화자 전환**). 나머지 0x3F1–0x3FB=특수명령(적음). 0x400+ 산발=비대사 구간 오디코드(무시).
- **단일 저제어**: `0x05`=개행(nl), `0x06/07 xx`=제어+인자, `0x08–0F`=기타 제어(어드벤처는 이들도 사용, 673과 달리), `0x00`=빈번한 씬 구분바이트(메시지끝 아님).
- **메시지 프레이밍**: `0x3F6`으로 분할하면 화자별 대화 턴(~341개/40KB)이 깔끔히 잡힘. 각 턴 = `[씬제어·화자셋업] [이름] 「 [대사 (05개행·3F5페이지) ...] 0x3F6`.
- ⚠️ **미해결(씬 인터프리터)**: 화자 셋업의 **일부 인자 바이트가 글리프값(こ=0x19·い=0x11·田 등 포트레이트/박스 파라미터)**이라 값만으로 "제어인자 vs 표시텍스트(이름)" 경계를 못 가름. 대사 본문(「이후)은 명확히 분리되나, **이름 시작 지점 확정엔 인터프리터 역공학 필요**. `CMP #$03EF`는 코드에 없음(데이터 오탐 2건 $C4:8B2D·$C6:1E4F) → 제어판별은 프리픽스 0x04 기반으로 추정. 인터프리터(출력버퍼 [$15] 소비자) 미확정.
- 도구: `adv_codec.py`(decompress·render), 렌더덤프 `tmp/adv/c5_dump.txt`.

## ⚠️ 2026-07-13 씬 로더 역공학 — 재삽입 난이도 확정

- **디코더 루틴 진입 = `$C0:39D5`** (JSL, 호출처 `$C0:3D0A`·`$C0:3D82`).
- 파라미터: 소스포인터 = **DP `$0A`(주소)+`$0C`(뱅크)**, 출력버퍼 `$7E:9DDE`(PEA #007E/#9DDE), A=길이/씬id 추정. 호출처가 스택으로 전달, 루틴이 $11-$13에 복사.
- **씬 소스는 게임 로직이 문맥적으로 설정** — `LDA #$5950` 같은 즉치 **없음**(0건). `STA $0A` 69곳·`STA $0C` 26곳에 분산. 즉 **정적 씬 테이블/즉치가 아니라 어떤 데이터구조에서 로드/계산** → 위치 미상.
- **재삽입 난점**: (1) 각 씬 소스는 **인게임 트레이스로만 확정**($C0:39D5에서 $0A/$0C 캡처 — `trace_advscene.lua`). (2) 한글은 일본어향 딕셔너리($C6:7C73)와 안 맞아 **압축률 나빠 원본 슬롯 초과 가능성 큼** → 재배치 필요. (3) 재배치하려면 씬 소스 셋업(위 미상 데이터구조)을 패치해야 하는데 위치 미상 → **관문**. (4) **유일한 손쉬운 경로 = 한글 재압축본이 씬 슬롯에 in-place로 맞는 경우**(짧은 씬은 가능성 있음).
- 결론: 어드벤처 재삽입은 **다관문 대형 작업**. 특정 씬 PoC = 인게임 트레이스로 씬 소스 확보 → 추출·번역 → (딕셔너리매칭) 압축기 제작 → **in-place 맞으면 삽입, 초과면 재배치 RE 추가 필요**.

## 동적 트레이스로 확정한 사실 (Mesen Lua)

파서 훅(`trace_text.lua`)으로 어드벤처를 플레이 → 파서 `$C1:9554`는 **어드벤처 대사를 미경유**(호출처 $C3789C·$D0516D 뿐). 즉 별개 엔진.

| 항목 | 값 | 근거 |
|---|---|---|
| **엔진(디코더 루프)** | **`$C0:39E0`~ (읽기 `$C0:39E7`, `$C0:3A05`, 딕셔너리 페치 `$C0:3A4A`, 출력 `$C0:3A4D`)** | `trace_c6.lua`, `trace_advptr.lua` |
| **소스 텍스트 뱅크** | **$C5**(주력, 최소 `$5950..$60C2`) · **$C4**(`$3429..$34D6+`) — 더 있을 것 | `trace_advptr.lua`(adv_ptr.txt) |
| **소스 포인터** | DP **`$11-$13`** 24비트 롱포인터 (`LDA [$11]`로 1바이트씩, `INC $11`/`$13`) | 디스어셈블 |
| **딕셔너리** | **`$C6:7C73`** (파서 글리프표로 보면 `　0123456789「」『』あいうえお…JKLM…！？。・…一九七十…` = charmap/프래그먼트 풀) | `trace_c6.lua`(c6_read.txt), 엔진 `LDA $7C73,X` |
| **출력 버퍼** | `STA [$15],Y` (DP `$15-$17` 롱포인터) → 디코드된 글리프 스트림 | 디스어셈블 |

### 압축(디코드) 개략
엔진 내부 루프(부분 디스어셈블, M/X 폭 토글로 정적 해석은 주의):
```
LDA [$11]        ; 소스값 A (16비트 추정) 읽고 $11 전진
... (제어 처리: $3A0D/$3A17 분기)
AND #$0007 ; INC ; STA $01   ; len = (A & 7) + 1
LSR LSR LSR ; TAX            ; X = A >> 3  (딕셔너리 인덱스)
loop $01회: LDA $7C73,X ; STA [$15],Y ; INX ; INY   ; 프래그먼트 확장
```
→ **소스 = (딕셔너리 인덱스, 길이) 참조 스트림**, 딕셔너리 `$C6:7C73`에서 (len)바이트 프래그먼트를 확장해 출력 버퍼에 씀. 사전 압축(dictionary/LZ 유사).
파서 글리프표로 소스를 직접 디코드하면 깨짐(단어 조각만 식별: ワールドグランプリ·チャンピオン·GPチップ·ディアナ·ビクトリーズ·われわれ).
