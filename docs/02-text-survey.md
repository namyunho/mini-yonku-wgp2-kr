# 02 · 텍스트 조사 (Text Survey)

초기 조사 2단계(텍스트 위치·인코딩). 도구: `scripts/survey.py`(SJIS 스캔·빈공간), `scripts/dump.py`(hex+SJIS 병렬 덤프). 리포트 원본: `tmp/survey_report.md`, `tmp/dump_report.txt`.

## 결론 (핵심)

| 질문 | 답 |
|------|-----|
| 본문 인코딩 | **표준 Shift-JIS** (커스텀 타일-인덱스 인코딩 아님) |
| 압축 | **비압축** — ROM에 SJIS 바이트 직접 저장 |
| 문자열 종결 | **널 바이트 `0x00`** |
| 공백 | 전각 공백 `81 40`(　) |
| 포인터 | **2바이트 리틀엔디언 동일-뱅크 포인터 테이블**이 각 문자열 블록 선두에 위치 |
| 위치 | 주로 뱅크 **$C0–$C1**(파일 0x00xxxx–0x01xxxx). $DB–$DC의 "한자" 밀집은 **폰트/그래픽 타일 오검출** |

→ 추출이 매우 유리하다. SJIS 디코더(`encoding_rs`)로 바로 읽고, 포인터 테이블로 문자열 경계를 확정한다. 한글화의 관건은 인코딩 역공학이 아니라 **폰트(SJIS 글리프 경로) 교체와 한글 코드포인트 매핑**으로 넘어간다(다음 단계).

## 확인된 텍스트 영역 (실측)

| 파일 오프셋 | SNES(HiROM) | 내용 |
|-------------|-------------|------|
| `0x0071B9` | $C0:71B9 | 타이틀 메뉴: はじめから / つづきから / うつす / けす / ほぞん / もどる |
| `0x00EB60` | $C0:EB60 | **2바이트 포인터 테이블** → 이어지는 머신명 문자열 |
| `0x00EB73` | $C0:EB73 | 머신명(카타카나): マーキュリー, ヴィーナス, アース, マーズ, ジュピター, サタン, ウラヌス, ネプチューン, プルート, ノヴァ … (행성명 미니카) |
| `0x00F480` | $C0:F480 | 2바이트 포인터 테이블 |
| `0x00F4B9` | $C0:F4B9 | 캐릭터명: せいば　レツ / せいば　ゴー / たかば　リョウ / みくに　とうきち / ブレット / エッジ / ミハエル … |
| `0x01C595` | $C1:C595 | UI 안내·다이얼로그: Ｘで　メニュー / Ａで　けってい / Ｂで　もどる / はい / いいえ / セーブ / ロード / いまのセッティングを　セーブしますか？ / すでに　データが　あります … |

문자열 사이사이에 타일맵/그래픽 데이터(예: `8F 8F 24 25 26…`, 창 테두리 `E0 E1…E6 E7`)가 섞여 있으므로, **추출은 추측 스캔이 아니라 포인터 테이블 기반**으로 해야 한다(누락·오추출 방지).

### 포인터 구조 예시 (0xEB60)

```
00EB60  FF 77 EB 84 EB 8F EB 96 EB 9D EB A8 EB AF EB B8 EB ...
             └$EB77┘└$EB84┘└$EB8F┘ ...   ← 2바이트 LE, 뱅크 $C0 내부 주소
00EB73: 83 7D 81 5B 83 4C 83 85 83 8A 81 5B 00  = マーキュリー\0
```

포인터 값은 뱅크 오프셋(addr)만 담고 뱅크는 참조 코드/DBR로 결정된다(동일-뱅크 테이블). 재배치 시 뱅크 내 free space 필요(플랫폼 노트 §6).

## round-trip 검증 (2단계 매체 게이트)

카트리지 flat ROM은 디스크 파일시스템·컨테이너가 없어 **전체 ROM 레벨 round-trip = 항등**이다. 도구가 ROM 전체를 읽어 무수정 재기록하면 바이트 동일(롬 이동 시 MD5 `acdeb2ee…` 일치로 이미 확인). 의미 있는 round-trip 검증은 이후 **폰트 타일 인코드/디코드**와 **포인터 테이블 파싱→재직렬화**에서 property 테스트로 수행한다(플랫폼 노트 §5).
