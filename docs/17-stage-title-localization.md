# 스테이지 1~10 시나리오 제목 한글화

## 결론

게임 시작 및 세이브 로드 직후 표시되는 제목은 정적 대사 `$C7/$C1/$D0`가 아니라
ROM `$C0:8234~$C0:82C8`의 별도 10개 문자열이다. `$C0:830F`의 첫 10개
16비트 포인터가 이 문자열들을 가리킨다. 문자열은 대사와 같은 `$CA` 전역
가변 길이 글리프 인코딩을 사용한다.

`assets/translations/stage_titles.json`을 정본으로 추가하고 `build_patch.py`가
번역문을 원래의 연속 149바이트 영역 안에 다시 패킹한 뒤 첫 10개 포인터를
갱신하도록 구현했다. stage 11 이후 데이터가 시작하는 `$C0:82C9`와 포인터
테이블의 나머지 항목은 변경하지 않는다.

## IDA 역추적 근거

- 제목 포인터 테이블: `$C0:830F` (`PC 0x00830F`)
- 제목 원문 영역: `$C0:8234~$C0:82C8` (`PC 0x008234~0x0082C8`)
- 호출 함수: `$C0:75D3`
- 핵심 흐름:

```text
$C0:75FE  LDA $7EC2F5
            CMP #$FFFF
            BNE $C0:7615
$C0:7615  PEA #$00C0
            LDA $7EC2F7       ; stage index
            ASL
            TAX
$C0:761E  LDA.l $C0830F,X    ; title pointer
            PHA
            LDA #$0018
            JSL $C06634       ; title renderer wrapper
```

`$C0:6634`는 전달된 bank/pointer를 렌더 상태에 넣고 `$C0:6660`을 호출한다.
후자는 간접 long pointer에서 종료자 포함 스트림을 렌더 버퍼로 복사한다.
세이브 로드 경로 `$C1:8988`은 세이브 메타데이터에서 stage 값을
`$7EC2F5/$7EC2F7`로 복원한다. 따라서 새 게임과 세이브 로드가 동일 테이블을
공유한다.

## 전역 글리프 결합과 해결

제목 음절에 1바이트 슬롯 우선권을 주는 최초 방식은 기존 `$CA` 전역 글리프
인덱스를 바꾸었다. 그 결과 정적 대사 6개가 원본으로 남고 어드벤처에서 긴 런
48개가 생겼다. 이 방식은 폐기했다.

최종 방식은 기존 할당 순서(정적 대사 → 어드벤처)를 완전히 보존하고 제목에만
새로 필요한 음절을 마지막에 추가한다. 개별 원본 슬롯을 고집하지 않고 이미
역추적한 10개 포인터를 갱신하여 149바이트 전체를 공동 사용한다. 최종 제목은
136/149바이트이며 기존 글리프 매핑과 대사 압축 결과가 기준선과 동일하다.
