# 20 · 장소별 필드/NPC 숨은 압축 레코드 전수 조사

> 2026-07-22 Codex 단독 조사. **실기 검증 전 커밋 금지** 상태다.

## 결론

연구소 NPC를 포함한 장소별 대사는 전역 씬표 `$C6:9C57`의 250개 큰 씬만으로 구성되지 않는다.
큰 씬 사이의 빈 구간과 마지막 씬 뒤에는 필드 오브젝트가 직접 참조하는 작은 압축 VM 레코드가
연속 저장돼 있다.

| 항목 | 결과 |
|---|---:|
| 씬간 숨은 팩 | 37곳 |
| 마지막 씬 뒤 꼬리 팩 | 1곳, `$C6:6B3C..6C84` |
| 숨은 압축 레코드 | **1,207개** |
| 뱅크별 | C4 484 / C5 290 / C6 433 |
| 압축 점유 | 76,604B, 잔여 0B |
| C2 포인터 참조 | **1,290개**, 모든 레코드가 1개 이상 보유 |
| C2 패턴 오탐 | **10개**(타일맵·일반 데이터, 패치 금지) |
| 엄격 VM 완주 | 1,206 / 1,207 |
| 텍스트런 후보 | 1,411런 / 고유 원문 1,340개 |

따라서 장소별 소스 주소는 서로 다르지만, 저장 규칙은 하나로 수렴한다. 레코드 경계와 참조 주소는
전량 카탈로그화할 수 있으며 `assets/translations/field_text.json`에 기록했다.

## 연구소 NPC에서 확인한 실제 경로

실기 추적으로 깨진 연구소 NPC 대사의 디컴프레서 입력이 `$C5:A781`임을 확인했다. 이 주소는
전역 씬 216의 압축 끝 `$C5:A688`과 다음 씬 217 `$C5:A8E1` 사이에 있는 10개 숨은 레코드 중 하나다.

`$C5:A781`을 독립 압축 해제하면 27B VM이 나오며 원문은 다음과 같다.

```text
ふじた
「ぼくのサタンはサイコーだろ？
```

인접한 `$C5:A6ED/$A73C/$A79B/$A803`에도 키시카와·하야시·마츠모토·츠치야 관련 독립 대사가
들어 있다. 즉 연구소 하나의 특수 예가 아니라, 각 장소 오브젝트가 자기 레코드를 C2 포인터로
고르는 공통 구조다.

## 완전성 증명

### 1. 전역 씬 저장영역의 연속 피복

- 첫 전역 씬은 `$C4:0000`에서 시작한다.
- 인접한 249쌍 중 212쌍은 앞 씬 압축 끝과 다음 씬 시작이 정확히 붙는다.
- 나머지 37쌍의 양수 갭 76,276B는 작은 압축 레코드 1,188개로 끝까지 정확히 소진된다.
- 음수 갭/압축영역 중첩은 0건이다.
- 마지막 씬 249의 끝 `$C6:6B3C` 뒤에는 C2가 참조하는 레코드 19개가 더 있고,
  `$C6:6C84`에서 정확히 끝난다.

따라서 `$C4:0000..$C6:6C84`의 어드벤처/필드 압축 저장영역에는 미분류 바이트가 없다.

### 2. 포인터 역검사

필드 포인터는 C2 데이터의 3바이트 형식이다.

```text
{addr_lo, addr_hi, bank-$C4}
```

1,207개 모든 레코드에서 이 형식의 C2 참조를 최소 1개 찾았다. 단, C2 전체의 단순 3바이트
패턴 검색은 타일맵의 `00 50 00` 같은 값을 `$C4:5000` 포인터로 오인한다. 중복 참조 레코드의
구조화된 포인터 표와 주변 데이터를 대조하고 Mesen CDL 데이터 접근 이력을 확인해 10개를
오탐으로 확정했다. 원시 일치 1,300개 중 실제 패치 대상은 1,290개다.

오탐 주소는 `$C2:0977/$1C9D/$5AEB/$87F3/$8E13/$92E0/$A20F/$A942/$CD47/$CD6D`다.
이들을 포인터로 재작성한 빌드는 월드맵 BG 타일과 일부 OBJ를 훼손했다. 따라서 카탈로그와
재삽입기는 이 10곳을 명시적으로 제외하며, 단순 패턴 일치 수와 실제 포인터 수를 별도로 검증한다.

꼬리 팩 뒤 `$C6:6C84..7C73`에서 C2의 모든 3바이트 창이 가리키는 19개 후보도 다시 압축 해제하고
엄격 VM으로 검사했다. 19개 모두 그래픽/일반 데이터로 판정돼 추가 필드 VM 후보는 0개였다.
`$C6:7C73..9C56`은 이미 확정된 어드벤처 압축 사전이므로 필드 레코드 영역의 끝도 고정된다.

## VM 워커 정정

기존 워커의 16개 desync를 ROM 핸들러와 대조해 두 가지 오해를 고쳤다.

1. `cmd 0x52`의 `$C0:52D1` 핸들러는 인자 1바이트를 읽고 커서를 1 올린다.
2. `$C0:5B4A` 표현식은 평면 목록이 아니다. `0x41/42/43...` 일부 서브옵이 다시 `$5B4A`를
   호출하는 재귀 prefix 문법이다.
3. `cmd 0x58`은 선형 디스어셈블상 표현식 호출이 7회 보이지만, 상호배타 분기 양쪽이 각 3회라
   직렬화된 입력은 첫 식을 포함해 4식이다.

엄격 모드에서는 1,206개가 끝까지 정렬된다. 유일한 `$C4:9B72`는 모든 텍스트런을 읽은 뒤
비실행 꼬리 `06 00 02 01 29 00 13 28 2F 00` 10B에서 멈추며, 꼬리에 텍스트 명령 `0x20/0x21`은
없다. 따라서 텍스트 누락은 없다.

기존 `adventure_kr.json`은 과거의 관대한 선형 워커가 만든 run 주소를 SSOT로 이미 사용한다.
기존 어드벤처 번역의 run 주소를 흔들지 않도록 `adv_scene.walk()` 기본값은 호환 모드로 유지하고,
새 필드 추출기만 `strict=True` 재귀 문법을 사용한다. `build_adv.py` 결과는 기존과 동일한
현재 통합 빌드는 231씬/1,724메시지/긴 런 0이다.

## 산출물

- `scripts/extract_field_text.py`: 원본 MD5, 전역 씬 연속성, 압축 경계, C2 참조, 꼬리 역검사,
  엄격 VM 문법을 회귀 불변식으로 검사한다.
- `assets/translations/field_text.json`: 38개 저장영역과 1,207개 레코드, 검증된 1,290개 포인터 위치와
  제외한 패턴 오탐 10개,
  텍스트 run 주소·raw·일본어와 번역 입력란을 담는다. `text_kr_full`은 축약 전 완역본,
  `text_kr`은 실제 위치보존 삽입용 문구로 분리하며 재추출해도 두 필드를 병합 보존한다.
- `scripts/adv_scene.py`: 실제 소비 규칙의 엄격 모드와 기존 번역 호환 모드를 분리한다.

## 다음 단계

레코드 발굴과 경계 확정은 끝났지만 1,340개 고유 원문의 번역·글리프 수요 집계·재압축·재배치·
C2 포인터 패치는 아직 하지 않았다. 원래 월드맵 퀴즈 예상량보다 훨씬 큰 별도 번역 블록이므로,
기존 글리프 인덱스를 보존한 append-only 할당과 전용 자유공간 설계를 먼저 확정한 뒤 재삽입한다.
원본보다 긴 런은 `text_kr_full`을 덮어쓰지 않고 `text_kr`만 축약하며, 별도 필드 축약 원장에
`before_kr→after_kr`와 바이트 상한을 영구 기록한다(docs/15 규약 계승).
